# Document Roles

## 역할 표

| 문서 종류 | 역할 | 포함해야 하는 것 | 포함하면 안 되는 것 |
|---|---|---|---|
| `AGENTS.md` | 부트스트랩과 라우팅 | 읽기 순서, 단일 소스 링크, 디렉터리 진입점 | 긴 배경 설명, 상세 명령, 상태표 |
| `docs/INDEX.md` | 문서 선택 가이드 | 작업 유형별 진입점 | 본문 규칙 재설명 |
| `docs/context-map.md` | 경로별 필독 문서 맵 | 경로와 필독 문서 매핑 | 각 문서의 장문 요약 |
| `ARCHITECTURE.md` | 레이어 불변식 | 의존 규칙, 책임, 핵심 판단 기준 | 구현 상태표, 전체 트리, 외부 라이브러리 장문 설명 |
| `workflow/agent-workflow.md` | 작업 절차 | 시작 순서, exec-plan 규칙, 검증 원칙 | 플랫폼별 상세 명령 |
| `workflow/verification.md` | 상세 검증 명령 | 빌드/테스트/검사 명령, 환경 메모 | 작업 절차 재설명 |
| `design/*` | 목표와 철학 | 제품 범위, 설계 원칙 | 경로별 라우팅 |
| `reference/*` | 참조 정보 | 전체 트리, 라이브러리 표, 보조 자료 | 필수 선독 규칙 |
| `exec-plans/active/INDEX.md` | 활성 계획 요약 | plan 이름, 영향 경로, 읽어야 하는 경우 | plan 본문 상세 복제 |

## 역할 판정 질문

- 이 문서가 없어도 다른 단일 소스 문서만으로 작업 시작이 가능한가?
- 이 내용은 규칙인가, 절차인가, 참조인가?
- 같은 내용이 두 문서 이상에 있으면 어느 쪽이 진짜 소유자인가?
