# 문서 존재 가치 감사 스킬

설계·철학 문서가 실제로 에이전트나 사람에게 가치를 제공하는지 주기적으로 점검한다.
수동 실행 전용. 리스트업만 하며 변경은 사람이 결정한다.

## 검사 대상

기본 대상:
- `docs/design/`
- `docs/conventions/`
- `server/docs/design/`
- `Lib/docs/` (설계 문서 한정)

## 존재 가치 판정 기준

| 기준 | 설명 |
|---|---|
| **로드 경로 외부** | AGENTS.md, context-map.md에서 아무도 링크하지 않음 |
| **내용 중복** | 다른 문서에 거의 같은 내용이 이미 존재 |
| **상위 문서와 모순** | CLAUDE.md, AGENTS.md와 충돌하는 규칙 포함 |
| **성격 불일치** | 철학/설계 문서인데 HOW-TO나 운영 절차가 혼입 |
| **에이전트 행동 영향 없음** | 추상적 원칙만 있고 도구나 검사로 강제 불가 |

하나라도 해당하면 후보로 올린다. 여러 기준에 해당할수록 삭제 쪽으로 기운다.

## 실행 절차

### 1단계: Claude 1차 분석

각 대상 문서에 대해:
1. 문서를 읽는다
2. 위 5개 기준을 체크한다
3. 링크 참조 현황을 확인한다 (`Grep`으로 파일명 검색)
4. 예비 판정 (삭제 후보 / 재작성 후보 / 유지)을 기록한다

### 2단계: 외부 리뷰 (선택)

판정이 불명확한 문서가 있으면 `opencode-review` 또는 `codex:rescue`로 2차 리뷰를 요청한다.
평가 기준을 동일하게 전달하고 결과를 통합한다.

### 3단계: 결과 출력

아래 형식으로 리스트를 출력한다. 파일은 저장하지 않는다.

```
## 삭제 후보
- path/to/doc.md
  - 이유: [해당 기준 명시]
  - 참조 파일: N개 (수정 필요)

## 재작성 후보
- path/to/doc.md
  - 이유: [해당 기준 명시]
  - 제안: [어떻게 고쳐야 하는지 한 줄]

## 유지
- path/to/doc.md — [간단한 이유]
```

## 외부 리뷰 연동 원칙

- Claude와 외부 리뷰가 동의 → 고신뢰 판정
- 외부 리뷰만 발견 → 재검토 필요
- Claude만 발견 → 유지 (외부 미발견이 반증은 아님)
- 충돌 → 양쪽 관점을 병기하고 사람에게 판단 요청

## 주의사항

- 결과는 리스트업으로 끝낸다. 파일 삭제, 수정, 참조 제거는 하지 않는다.
- 삭제 후보라도 사람의 확인 없이 진행하지 않는다.
- 참조 파일 수가 많을수록 영향 범위를 함께 명시한다.
