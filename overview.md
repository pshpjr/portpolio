# 프로젝트 개요 (overview.md)

## 목표

2~4인 인스턴스 던전 RPG 게임 서버 포트폴리오 프로젝트.
"게임 서버 엔지니어로서 프로덕션 수준의 아키텍처를 설계하고 구현할 수 있다"는 것을 증명하는 것이 목적이다.

---

## 핵심 기술 목표 (반드시 구현)

| 목표 | 설명 |
|------|------|
| 서버사이드 히트 판정 | 클라이언트 신뢰 불가. 서버에서 충돌/범위 계산 |
| 데이터 드리븐 설정 | 스킬/몬스터/아이템을 코드 수정 없이 JSON으로 정의 |
| 런타임 핫리로드 | 서버 재시작 없이 데이터 파일 변경 반영 |
| 인스턴스 격리 | 던전 인스턴스마다 독립적인 상태 관리 |
| 서버 권위적 전투 | 데미지, 쿨다운, 사망 판정 모두 서버에서 결정 |

---

## 범위 (Scope)

**포함:**
- 게임 서버 (C++)
- 데이터 정의 파일 (JSON / FlatBuffers)
- 서버 테스트 클라이언트 (간단한 CLI 또는 Python)
- 빌드/검사 툴 (Python)

**포함하지 않음 (Non-scope):**
- 그래픽 클라이언트 (Unity 등)
- 실제 서비스 배포 인프라
- 계정/인증 시스템 (더미 처리)
- DB 연동 (인메모리로 처리)

---

## 플레이어 경험 흐름 (서버 관점)

```
클라이언트 접속
    → 로비 입장 (세션 생성)
    → 파티 구성 (2~4인) 또는 솔로
    → 매칭/던전 인스턴스 생성
    → 던전 플레이 (전투, 이동, 아이템)
    → 던전 클리어 or 실패
    → 결과 처리 → 로비 복귀
```

---

## 성공 기준

- [ ] 2인 이상 동시 접속 던전 플레이 가능
- [ ] 히트 판정이 서버에서 이루어짐 (클라이언트 조작 불가)
- [ ] 데이터 파일 수정 후 서버 재시작 없이 반영됨
- [ ] 단위 테스트 커버리지 Core 레이어 60% 이상
- [ ] 레이어 의존성 위반 0건 (CI 통과)

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 언어 | C++17 | 게임 서버 업계 표준 |
| 빌드 | CMake | 크로스 플랫폼 |
| 네트워크 | Asio (standalone) | 헤더 온리, 가볍고 실용적 |
| 직렬화 | FlatBuffers | 성능, 스키마 명시 |
| 설정 파싱 | nlohmann/json | 직관적인 API |
| 로깅 | spdlog | 성능, 비동기 지원 |
| 테스트 | GoogleTest | 업계 표준 |
| 툴 | Python 3.10+ | 코드 생성, 검사, 자동화 |
