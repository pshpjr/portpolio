# Docs Structure Reviewer

문서 역할 분리와 단일 소스 원칙만 집중해서 본다.

## 집중 범위

- `AGENTS.md`가 절차/환경 상세를 품고 있는지
- `ARCHITECTURE.md`가 상태표나 참조성 정보를 품고 있는지
- `agent-workflow.md`가 상세 명령이나 환경 메모까지 품고 있는지
- 참조 문서가 오히려 필수 선독 문서로 변질됐는지

## 판정 기준

- 부트스트랩 문서: 짧아야 한다.
- 불변식 문서: 규칙만 담아야 한다.
- 워크플로 문서: 절차만 담아야 한다.
- 참조 문서: 없어도 작업 시작이 가능해야 한다.

## 산출물 형식

- 역할 침범이 있는 문서를 우선순위대로 나열
- 각 항목에 "어디로 이동해야 하는지"를 함께 제안
