# 파티 시스템

## 목적

1~4인 협동 플레이를 위한 파티 생성, 초대, 유지, 던전 연계를 정의한다.

## 핵심 규칙

- 파티는 최대 4인까지 구성한다.
- 솔로 입장도 내부적으로는 1인 파티로 처리한다.
- 파티장은 초대, 추방, 파티장 위임, 던전 입장 요청 권한을 가진다.
- 던전 진행 중에는 파티 탈퇴나 해산을 허용하지 않는다.
- 공개 파티만 파티 찾기 목록에 노출한다.
- 파티 채팅은 마을과 던전 모두에서 유지된다.

## 주요 흐름

### 파티 생성과 초대

1. 플레이어가 공개 또는 비공개 파티를 생성한다.
2. 생성자는 자동으로 파티장이 된다.
3. 파티장은 초대를 보내거나 공개 파티 지원자를 받는다.
4. 수락 시 파티원 목록이 갱신되고 전체 파티에 상태가 전파된다.
5. 정원이 차면 추가 초대와 가입을 막는다.

### 파티 유지

1. 파티원은 자유롭게 탈퇴할 수 있다.
2. 파티장이 나가면 다음 순서의 파티원에게 리더 권한이 이전된다.
3. 파티원이 모두 없어지면 파티를 해산한다.
4. 접속이 끊긴 파티원은 60초 동안 복귀를 기다린다.
5. 유예 시간 내 복귀하지 않으면 자동 추방한다.

### 던전 연계

1. 파티장만 던전 입장을 시작할 수 있다.
2. 입장 준비가 끝나면 파티 상태가 `READY`를 거쳐 `IN_DUNGEON`으로 전이된다.
3. 던전 중에는 파티를 유지한 채 진행한다.
4. 완료 후 `COMPLETED` 또는 필요 시 `DISBANDED` 상태로 마무리한다.

## 제약과 예외

- 솔로 입장은 1인 파티 자동 생성으로 처리하며, 별도 예외 흐름을 최소화한다.
- 공개 파티 찾기 기능은 유지하되 세부 필터링과 자동 매칭은 현재 스코프 밖이다.
- 파티 상태 전이는 `FORMING -> READY -> IN_DUNGEON -> COMPLETED / DISBANDED`를 기본으로 한다.
- 세션, 레코드, 브로드캐스트 방식 같은 구현 세부는 앱 문서에서 확정한다.
