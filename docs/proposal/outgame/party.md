# 파티 시스템

> 관련 테이블: `Party`, `PartyMember`

## 개요

1~4인 파티를 구성하고 던전 입장까지 연결되는 시스템.
솔로 입장 시에도 1인 파티가 자동 생성되어 동일한 흐름으로 처리된다.

## 파티 생성 / 초대 흐름

1. 파티장이 파티 생성 요청 (공개/비공개 선택)
2. 서버: Party 레코드 생성 (status = FORMING), 파티장 자동 참가
3. 파티장이 다른 플레이어에게 초대 요청 전송
4. 초대 받은 플레이어 수락/거절
5. 수락 시: PartyMember 추가, 파티원 전체에 파티 정보 갱신 브로드캐스트
6. 거절 시: 파티장에게 거절 통보
7. 현재인원 == max_party일 때 초대/가입 불가

## 파티 찾기

- 파티장이 생성 시 공개/비공개를 선택
- 공개 파티는 파티 찾기 목록에 표시
  - 표시 정보: 파티명, 현재인원/최대인원, 목표 던전
- 공개 파티에 가입 신청 -> 파티장이 수락/거절

## 파티장 권한

- 초대: 다른 플레이어에게 파티 초대
- 추방: 파티원 강제 추방
- 파티장 이전: 다른 파티원에게 파티장 위임
- 던전 입장 요청: 파티장만 던전 입장 시작 가능

## 파티 해산 / 탈퇴 흐름

- 파티원 탈퇴: PartyMember 제거, 파티원 전체에 갱신 브로드캐스트
- 파티장 탈퇴: 다음 입장 순서(PartyMember 등록 순) 멤버에게 파티장 자동 이전. 파티원이 없으면 해산
- 던전 진행 중 파티 탈퇴/해산: 불허. 던전 내에서는 파티를 유지해야 함

## 파티원 오프라인 처리

- 접속 끊김 시 60초 유예
- 60초 내 재접속하면 파티 복귀
- 60초 초과 시 자동 추방

## 솔로 입장

- 파티 없이 던전 입장 요청 시 1인 파티 자동 생성
- 동의 창 스킵, 기존 흐름과 동일하게 처리

## 파티 채팅

- 파티원 전용 채팅 채널 (채팅 시스템 참조)
- 파티 채팅 메시지는 파티원에게만 브로드캐스트
- 마을과 던전 모두에서 사용 가능

## 파티 상태 전이

```
FORMING -> READY (던전 입장 동의 완료) -> IN_DUNGEON -> COMPLETED / DISBANDED
```
