# 채팅 시스템

## 목적

마을과 파티 플레이에 필요한 최소 채널만 제공해 커뮤니케이션 비용을 낮춘다.

## 핵심 규칙

- 채널은 `GLOBAL`과 `PARTY` 두 종류만 지원한다.
- 채팅은 서버가 중계하며 영구 저장하지 않는다.
- 던전에서는 파티 채팅만 실질적으로 사용한다.
- 메시지 길이는 200자로 제한한다.
- 같은 유저는 1초 안에 연속 메시지를 보낼 수 없다.
- 텍스트 채팅 외에 짧은 빠른 핑을 지원한다.

## 주요 흐름

1. 클라이언트가 채널 타입과 메시지를 서버에 전송한다.
2. 서버가 길이와 쿨다운을 검사한다.
3. 서버가 채널에 맞는 대상 세션을 계산한다.
4. 대상 세션에 메시지를 브로드캐스트한다.
5. 클라이언트가 채팅 UI에 출력한다.

### 빠른 핑

- 최소 핑 세트는 `준비`, `집결`, `분산`, `부활 요청`, `따라와`, `인터럽트` 6종으로 제한한다.
- 빠른 핑은 던전과 READY 구간에서 사용 가능하다.
- 빠른 핑은 짧은 아이콘과 1줄 텍스트만 보여주며, 음성 채팅 대체 기능으로 확장하지 않는다.
- 시스템 알림은 `입장 거절`, `미응답`, `재접속 카운트다운`, `체크포인트 도달`을 우선 노출한다.

## 제약과 예외

- `GLOBAL`은 마을 플레이어 기준 채널로 유지한다.
- `PARTY`는 마을과 던전에서 모두 사용할 수 있다.
- 귓속말, 필터링, 로그 저장은 현재 포트폴리오 범위 밖이다.
- 패킷 필드와 직렬화 구조는 구현 문서에서 확정한다.
