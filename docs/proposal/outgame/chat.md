# 채팅 시스템

## 개요

전체 채팅(GLOBAL)과 파티 채팅(PARTY) 두 채널을 지원.
채팅은 서버가 중계하며 DB에 저장하지 않는다 (메모리에서만 처리).

## 채팅창 UI

- 탭 2개: 전체(GLOBAL) / 파티(PARTY)
- 던전에서는 파티 탭만 활성화

## 메시지 규칙

- 메시지 길이 제한: 200자
- 도배 방지: 같은 유저 1초 쿨다운 (1초 내 연속 메시지 서버에서 무시)
- 귓속말 채널 없음 (포트폴리오 스코프 외)
- 채팅 필터 없음 (포트폴리오 스코프 외)

## 메시지 흐름

1. 클라이언트 -> 서버: 채팅 메시지 패킷 (channel_type, content)
2. 서버: 쿨다운 검사, 메시지 길이 검증
3. 서버: 채널 타입에 따라 대상 세션 목록 조회
   - GLOBAL: 마을에 있는 전체 플레이어 세션
   - PARTY: 동일 파티원 세션
4. 대상 세션 전체에 브로드캐스트 패킷 전송
5. 클라이언트 수신 -> UI 출력

## 메시지 구조 (패킷)

```
ChatMessage {
  channel_type: enum (GLOBAL / PARTY)
  sender_id: int
  sender_name: string
  content: string (최대 200자)
  timestamp: int64
}
```
