# 던전 입장 시스템

## 목적

파티 구성 이후 던전 입장, 보상 모드 선택, 전원 동의 절차를 정의한다.

## 핵심 규칙

- 던전 입장은 파티장만 시작할 수 있다.
- 입장 전 보상 모드를 선택해야 한다.
- 입장 전 현재 무기 프리셋과 전리품 선호 무기를 확정해야 한다.
- 전원 동의가 있어야 인스턴스를 생성한다.
- 제한형 모드는 고가치 보상을 주지만 횟수 제한이 있다.
- 무제한형 모드는 반복 입장이 가능하지만 보상 풀이 축소된다.
- 입장 시 파티 인원수에 따라 기믹 난이도와 적 수치가 조정된다.

플레이어 표시 이름:
- `제한형 모드` = `정화 원정`
- `무제한형 모드` = `정찰 원정`

## 주요 흐름

1. 파티장이 입구에서 던전 입장을 요청한다.
2. `정화 원정` 또는 `정찰 원정`을 선택한다.
3. 각 파티원이 현재 무기 프리셋과 전리품 선호 무기를 확인한다.
4. 서버가 각 파티원의 보상 횟수와 입장 가능 여부를 확인한다.
5. 전원에게 동의 팝업과 차단 사유 요약을 전송한다.
6. 15초 안에 전원이 수락하면 인스턴스를 생성하고 던전으로 이동한다.
7. 거절, 미응답, 조건 미충족자가 있으면 입장을 취소하고 파티장에게 재시도 권한을 남긴다.

## 입장 UI와 확인 항목

- 파티원 목록에 `준비`, `미응답`, `차단 사유`를 함께 표시한다.
- 차단 사유는 `횟수 부족`, `장비 수리 필요`, `무기 프리셋 미확정`처럼 짧은 문구로 노출한다.
- 전원 수락 직전 화면에는 현재 원정 모드, 무기 프리셋, 전리품 선호 무기를 다시 보여준다.
- 정찰 원정은 연습/저강도 파밍용이라는 점을 UI 문구로 명확히 표시한다.

## 제약과 예외

- 솔로 입장은 1인 파티 생성 이후 같은 흐름을 사용한다.
- 제한형 모드는 일일 또는 주간 횟수 제한을 소모한다.
- 무제한형 모드는 연습과 재료 파밍용으로 설계하며 고등급 아이템을 드롭하지 않는다.
- 실제 인스턴스 스레드 할당과 보상 수치 계산은 구현 문서에서 확정한다.
- 던전 입장 후에는 무기 전환, 스킬 프리셋 변경, 전리품 선호 변경을 허용하지 않는다.
