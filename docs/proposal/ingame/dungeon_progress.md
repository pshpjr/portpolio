# 던전 진행 관리

## 1. 개요
본 던전 시스템은 파티 규모에 맞춘 기믹과 **공정하고 독립적인 보상** 시스템을 지향한다. 또한 미래의 확장성을 고려하여 **모듈형 구역 설계**를 채택한다.

## 2. 구역(Zone) 및 기믹 구조
던전은 미래의 **절차적 던전 생성(PCG)** 도입을 고려하여 **방(Room) 기반 모듈형 구조**로 설계되었다.
- **구역 구성 (포트폴리오 버전)**: 현재는 1~3구역(숲길, 제단, 보스방)이 고정된 위치에 배치된 **단일 고정 맵**을 사용하여 레벨 디자인의 명확성과 구현 속도를 우선한다.
- **관문(Gate)**: 각 구역 사이에는 관문이 존재하며, 해당 구역의 몬스터를 모두 처치하거나 기믹을 해결해야 다음 공간으로의 문이 열린다.
- **역행 불가**: 이전 공간으로의 역행을 차단하여 서버 부하를 방지하고 인던 진행을 선형적으로 유지한다.
- **기믹 구성**: 협동 발판, 순차 레버 등 파티원 수(1~4인)에 따라 개수가 조절된다.
- **기믹 실패 구조**: 기믹은 무한히 실패할 수 없도록 설계되어 서버 부하를 방지한다.
  - **단발성 실패**: 특정 기믹 조작 실패 시, 즉시 **실패 페널티 페이즈**로 전환된다.
  - **중복 실패 방지**: 이미 기믹에 실패하여 소환된 정예 몬스터가 존재할 때는 추가 실패가 발생하지 않거나, 기믹이 비활성화된다.

## 3. 기믹 실패 시 페널티: 시간 지연
기믹 해결 실패 시 발생하는 리스크를 통해 공략의 긴장감을 높인다.
- **정예 몬스터 소환**: 기믹 실패 시 즉시 강력한 **방해 소환수** 그룹이 등장한다.
- **무보상 규칙**: 소환된 정예 몬스터는 **경험치와 아이템을 전혀 드랍하지 않는다.**
- **시간 소모**: 이들을 모두 처치할 때까지 다음 구역으로 넘어갈 수 없어, 전체 공략 시간이 지연되고 보상 획득 기회가 줄어드는 간접 패널티를 준다.
- **그로기 게이지 보유**: 페널티 소환수는 보스와 유사한 그로기 게이지를 가지며, 무력화로 빠르게 처치하는 것이 시간 단축의 핵심이다.
- **중복 소환 제한**: 서버 부하를 고려하여 한 번에 활성화되는 페널티 몬스터 수는 일정 수준으로 유지된다.

## 4. 드랍 및 보상 (Rewards)

> 관련 테이블: `Reward`, `RewardItem`, `Item`, `Inventory`

### 4.1 드랍 처리 흐름
1. 몬스터 사망 이벤트 발생
2. 서버: `Monster.reward_id` → `Reward` → `RewardItem` 목록 조회
3. 각 `RewardItem`에 대해 `drop_rate` 기반 독립 확률 계산 (합계 100% 제한 없음)
4. 드랍 결정된 아이템: `quantity_min ~ quantity_max` 범위에서 수량 결정
5. 파티원 각자에게 **개별적으로 드랍 판정** 수행
6. 드랍 아이템을 월드 오브젝트로 스폰 (각자 자기 것만 보임)
7. 클라이언트에 드랍 아이템 정보 전송

### 4.2 드랍 세부 규칙
- **개인 드랍**: 파티원 각각은 자신의 전리품만 획득. 타인의 전리품은 서버 권위에 의해 노출 및 획득이 제한된다. (파티 분쟁 원천 차단)
- **소환몹**: 드랍 없음 (`reward_id = null`)
- **자동 줍기(Auto Loot)**: 기본 활성화. 범위 내 진입 시 자동 획득. 드랍 아이템은 60초 후 소멸.
- **골드 드랍**: `RewardItem` 테이블에서 `item_id = 0`으로 GOLD 특수 처리.
- **장착 무기 보정**: 각 유저의 장착 무기 타입에 맞는 아이템의 드랍 확률이 우선 배정된다.

### 4.3 보스 드랍
- 확정 드랍(`drop_rate = 1.0`) 아이템과 확률 드랍 아이템이 혼합된다.
- 보스 처치 2초 후 **보물상자 오브젝트** 스폰 → 상호작용 시 개인 보상 UI 표시.
- 무력화 2회 이상 달성 시 **추가 보상 상자** 1개 지급.

### 4.4 보상 이원화
- **제한형 모드**: 일일/주간 횟수 제한. 에픽/레어 장비 및 희귀 스킬북 획득 가능.
- **무제한형 모드**: 횟수 제한 없음. 기본 재료·소량 경험치·골드만 지급. 고등급 아이템 드랍 없음.

## 5. 미래 비전: 절차적 던전 생성 (PCG)
- **확장 설계**: 각 구역(Room)의 타일셋(Tileset)을 무작위로 조합하고 연결하는 알고리즘을 도입하여, 매 입장 시마다 지형과 몬스터 배치가 달라지는 무한 던전 시스템으로의 확장을 고려하여 설계되었다.
- **데이터 구조**: 모든 몬스터 및 기믹의 배치 좌표는 해당 구역(Room)의 로컬 좌표계로 관리하여 공간 재배치가 용이하도록 한다.
