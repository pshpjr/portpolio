# 던전 진행 관리

## 목적

인던의 선형 진행, 협동 기믹, 페널티 구조, 보상 흐름을 일관된 규칙으로 정의한다.

## 핵심 규칙

- 포트폴리오 버전은 고정된 단일 던전 맵을 사용한다.
- 던전은 여러 구역으로 나뉘며 각 구역은 몬스터 처치나 기믹 해결 후 다음 구역으로 열린다.
- 역행은 제한하여 진행을 선형으로 유지한다.
- 협동 기믹은 파티 인원수에 맞춰 스케일링된다.
- 기믹 실패는 즉시 시간 지연형 페널티로 연결된다.
- 페널티 몬스터는 보상을 주지 않는다.
- 보상은 개인 드롭을 기본으로 하며, 보스 보상과 메인 보상 모드는 별도로 설계한다.

## 주요 흐름

### 던전 진행

1. 파티가 던전에 입장한다.
2. 현재 구역의 몬스터 처치 또는 기믹 해결을 수행한다.
3. 조건을 만족하면 관문이 열리고 다음 구역으로 이동한다.
4. 보스 구역까지 이 흐름을 반복한다.

### 기믹 실패

1. 플레이어가 협동 기믹 해결에 실패한다.
2. 실패 패널티 페이즈가 즉시 활성화된다.
3. 보상 없는 정예 페널티 몬스터가 소환된다.
4. 페널티 몬스터를 정리해야 다시 진행할 수 있다.

### 보상 처리

1. 몬스터 처치 시 각 파티원에게 개별 드롭 판정을 수행한다.
2. 장착 무기 타입에 맞는 드롭이 우선 보정된다.
3. 보스 처치 후에는 개인 보상 상자와 추가 보상 조건을 확인한다.
4. 제한형 또는 무제한형 보상 모드에 따라 최종 보상 풀이 달라진다.

## 던전 구조 (v1 고정 맵)

- 던전은 총 4개 구역으로 구성된다:
  1. **입구 구역**: 일반 몬스터 2~3그룹, 이동 튜토리얼용
  2. **중간 구역 A**: 정예 몬스터 포함, 첫 협동 기믹 존재
  3. **중간 구역 B**: 정예 + 일반 혼합, 두 번째 협동 기믹 존재
  4. **보스 구역**: 주술사 보스전

- 각 구역은 이전 구역의 모든 몬스터 처치 또는 기믹 해결 후 개방된다.
- 체크포인트는 구역 2, 3 진입 시 자동 저장된다 (구역 1과 보스 구역은 체크포인트 없음).

## 협동 기믹 규칙

### 기믹 유형 (v1 정의)

| 기믹 | 설명 | 인원 스케일링 |
|---|---|---|
| 제단 동시 발동 | N개 제단을 동시에 밟아 활성화 | 1인=1개, 2인=2개, 3~4인=2~3개 |
| 수호자 인도 | 특정 NPC를 적에게 맞지 않게 목표 지점으로 유도 | 인원과 무관, 난이도는 적 수로 조정 |

- v1에서는 위 2종 기믹만 구현한다.
- 기믹 성공 시 EXP 30 + 구역 진행 조건 충족.
- 기믹 실패 조건: 제단 기믹은 활성화 유지 시간(10초) 내 동시 점유 실패, 수호자 기믹은 NPC HP 0.

## 보상 구조

| 보상 종류 | 획득 조건 | 제한형 | 무제한형 |
|---|---|---|---|
| 일반 드롭 | 몬스터 처치 | 전 등급 | 노멀~언커먼만 |
| 보스 보상 상자 | 보스 처치 | 레어~에픽 포함 | 노멀~언커먼만 |
| 추가 보상 상자 | 무력화 2회 이상 달성 | 레어 1개 보장 | 없음 |
| 던전 클리어 EXP | 보스 처치 | 100 EXP | 50 EXP |

## 제약과 예외

- 현재 스코프는 고정 맵이지만, 구조는 향후 방 기반 절차 생성으로 확장 가능하게 유지한다.
- 이미 페널티 소환이 활성화된 상태에서는 중복 실패를 누적하지 않는다.
- 소환 몬스터는 경험치와 아이템을 드롭하지 않는다.
- 자동 줍기를 기본 전제로 하되, 상세 드롭 테이블과 오브젝트 처리 방식은 구현 단계에서 확정한다.
- 무력화 2회 이상 달성 보너스처럼 보스 추가 보상 조건은 보스 문서와 함께 조정한다.
