# 몬스터 AI 및 기믹 시스템

## 목적

일반 몬스터, 페널티 몬스터, 보스가 서로 다른 전투 압박을 만들도록 AI 역할을 정의한다.

## 핵심 규칙

- 일반 몬스터는 단순하고 읽기 쉬운 압박을 담당한다.
- 페널티 몬스터는 보상 없는 시간 지연 장치다.
- 보스는 위협도, 무력화, 카운터 같은 고급 전투 규칙을 활용한다.
- 특정 기믹 구간에서는 위협도와 무관한 강제 타게팅이 발생하며, 종료 시 위협도 1위를 즉시 재계산한다.
- 서버는 몬스터의 위치, 타게팅, 패턴 결정을 권위 있게 처리한다.

## 주요 흐름

### 일반 몬스터

1. 먼저 감지하거나 자신을 공격한 플레이어를 우선 추적한다.
2. 기본 공격과 단순 패턴으로 공간 압박을 만든다.
3. 대부분의 플레이어 공격에 경직과 넉백 반응을 보인다.

### 페널티 몬스터

1. 기믹 실패 시 즉시 소환된다.
2. 일반 몬스터보다 오래 버티며 공략 시간을 지연시킨다.
3. 보상을 주지 않고 순수한 리스크로만 작동한다.

### 보스 AI

1. 위협도 테이블을 기준으로 기본 타깃을 선택한다.
2. 페이즈 전환이나 특정 패턴에서 강제 타게팅을 사용할 수 있다.
3. 무력화와 카운터 성공 여부에 따라 패턴 강도와 빈틈이 달라진다.
4. 체력 구간에 따라 별도 기믹 또는 전멸기 패턴을 연다.

## 타깃 선택 규칙

| 규칙 | 값 |
|---|---:|
| 위협도 재평가 주기 | 0.5초 |
| 타깃 교체 조건 | 현재 타깃보다 위협도 15% 초과 |
| 최근 피격 보너스 유지 시간 | 2초 |
| 최근 피격 보너스 | +40 |

- 일반 몬스터는 위협도 시스템 대신 최근 공격자 우선 규칙을 사용한다.
- 보스와 정예만 위협도 테이블을 유지한다.

## 상태별 행동 제한

| 상태 | 제한 |
|---|---|
| `DOWNED` | 이동, 회전, 시전 불가 |
| `RECOVERY` | 무력화 게이지 감소 무효 |
| `FORCED_TARGET` | 위협도 무시, 지정된 타깃만 추적 |

## 일반 몬스터 FSM

| 상태 | 전환 조건 | 행동 |
|---|---|---|
| `IDLE` | - | 제자리 대기 또는 고정 패트롤 |
| `PATROL` | IDLE 진입 후 3초 | 지정 경로를 반복 순찰 |
| `DETECT` | 감지 범위 내 플레이어 발견 | 경보음, 타깃 설정 |
| `CHASE` | DETECT 직후 | 타깃 플레이어를 추격 |
| `ATTACK` | 공격 범위 내 진입 | 기본 공격 또는 패턴 실행 |
| `STAGGER` | 경직 피격 | 현재 행동 취소, 짧은 경직 |
| `DEAD` | HP 0 | 사망 처리, 드롭 계산 |

- `DETECT` 범위: 8m (기본값). 시야 기반이 아닌 원형 범위로 처리한다.
- `ATTACK` 범위: 1.5m (근접형), 8m (원거리형).
- 플레이어에게 공격받으면 `IDLE`/`PATROL` 상태에서도 즉시 `DETECT`로 전환한다.

## 몬스터 역할 분류

| 분류 | 특성 | 대표 행동 |
|---|---|---|
| 근접 돌진형 | 빠른 이동, 낮은 HP | 직선 돌진 후 기본 공격 |
| 근접 중형 | 표준 체력, 중간 딜 | 기본 공격 2~3타 연계 |
| 원거리 투사체형 | 먼 거리 유지 | 투사체 발사, 플레이어 접근 시 후퇴 |
| 보조형 | 낮은 딜, 높은 HP | 주변 몬스터 강화 버프 |
| 페널티 몬스터 | 정예급 체력, 드롭 없음 | 범위 공격 중심, 경직 저항 있음 |

- 포트폴리오 v1 기준 각 역할 최소 1종씩 구현을 목표로 한다.
- 페널티 몬스터는 별도 AI 분기 없이 보조형 기반에 드롭 비활성화로 처리한다.

## 제약과 예외

- 보스의 상세 패턴 값과 페이즈별 기술은 별도 보스 문서에서 관리한다.
- 보스는 특정 상태 이상에 면역이거나 반복 제어에 대한 저항을 가진다.
- 서버 틱 단위 동기화와 세부 FSM 구조는 구현 문서에서 확정한다.
