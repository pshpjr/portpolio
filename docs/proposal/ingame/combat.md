# 전투 시스템

## 목적

패링의 긴장감, 무기별 정체성, 보스 무력화 공략을 핵심으로 하는 협동 액션 전투를 정의한다.

## 1차 핵심 축

- 포트폴리오 기준 1차 전투 코어는 `무기 정체성`, `패링/카운터`, `무력화/다운`이다.
- 플레이어는 액티브 스킬 4개, 아이덴티티 스킬 1개, 소모품 2슬롯, 이동기 1개를 사용한다.
- 아이덴티티 게이지는 무기별 전투 리듬을 만드는 핵심 자원이다.
- 보스전의 핵심 재미는 전조 대응, 패링 성공, 무력화 타이밍 집중에 있다.
- 전투 판정은 서버 권위를 기본으로 하며, 1차 구현에서는 전투 판정의 일관성과 재현성을 우선한다.

## v1 Hands-First Loop

v1 전투는 매 몇 초마다 아래 3가지를 반복하게 만들어야 한다.

1. 지금 `패링/회피`로 받을지, 한 박자 기다릴지 판단한다.
2. 지금 `무력화 누적`을 노릴지, 안전한 기본 딜을 유지할지 판단한다.
3. 지금 `핵심 스킬을 털어 빈틈을 벌릴지`, 다음 큰 전조를 위해 아낄지 판단한다.

속성, 위협도, 내구도는 이 3가지 판단을 돕는 보조 시스템이며, 같은 비중의 핵심 축으로 취급하지 않는다.

## 카메라와 화면 구성

### 카메라

- 권장 시점은 `쿼터뷰 2.5D` 기준의 반고정 카메라다.
- 기본 카메라는 플레이어를 화면 하단 35~40% 지점에 두고, 전방 정보를 후방보다 더 넓게 보여준다.
- 기본 전투 카메라 체감 거리는 플레이어 기준 `전방 14m 내외`, `후방 6m 내외`가 읽히는 수준으로 잡는다.
- 카메라 각도는 `위에서 약 35~40도` 내려다보는 구도를 기준으로 한다.
- 보스전에서는 줌 인보다 `전조 가독성 유지`를 우선하며, 패턴이 커질수록 카메라를 소폭만 후퇴시킨다.

### 전투 HUD

- 화면 하단 중앙: 기본 공격, `Q/E/R/F`, 이동기, 아이덴티티, 소모품 쿨다운
- 화면 좌측: 플레이어 HP/자원, 파티원 HP, 전투 불능/부활 가능 여부
- 화면 상단: 보스 HP, 무력화 게이지, 인터럽트 가능 상태, 광폭화 타이머
- 화면 중앙 근처: 현재 대상 방향, 짧은 조준 보조, 패링 성공/실패 피드백
- 화면 우측 또는 상단 우측: 현재 구역 목표, 준비 상태, 기믹 진행도

### 전투 가독성 원칙

- 플레이어가 즉시 봐야 하는 정보는 `내 캐릭터`, `보스 전조`, `파티원 위험 상태` 순으로 우선한다.
- 보스의 위험 전조는 발밑 장판, 시전 바, 상단 상태 바 중 최소 2곳에서 중복 표현한다.
- 패링 성공, 인터럽트 성공, 무력화 진입은 화면 중앙 근처에 짧고 강한 피드백으로 준다.
- 화면 장식보다 판정 정보가 먼저 읽혀야 하며, VFX는 보스 전조를 가리지 않게 제어한다.

### 즉시 피드백 규격

| 이벤트 | 위치 | 지속시간 | 오디오 | 우선순위 |
|---|---|---:|---|---|
| 패링 성공 | 화면 중앙 하단 | 500ms | 짧은 금속 반향 | 높음 |
| 패링 실패 | 화면 중앙 하단 | 350ms | 둔탁한 실패음 | 중간 |
| 인터럽트 성공 | 화면 중앙 | 700ms | 끊기는 파열음 | 높음 |
| 무력화 진입 | 보스 상단 + 화면 중앙 | 1200ms | 저음 충격음 | 최상 |
| 광폭화 60초 전 경고 | 화면 상단 타이머 | 1000ms 점멸 | 경고 북소리 | 높음 |
| 파티원 전투 불능 | 좌측 파티 UI | 상태 지속 | 짧은 경고음 | 높음 |

### 파티 위험 상태 규칙

- 파티원 HP가 30% 미만이면 파티 UI 체력바를 적색 점멸로 바꾼다.
- 전투 불능 상태는 파티 UI와 월드 마커 양쪽에 표시한다.
- 부활 가능 대상은 거리와 남은 보호 시간을 함께 보여준다.
- 회복/부활 요청 핑이 오면 해당 파티원 UI에 2초간 강조 효과를 준다.

### 시각 효과 제어

- 보스 중심부와 플레이어 주변 2m 반경을 가리는 스크린 스페이스 VFX는 반투명 처리한다.
- 장판 전조 위에 추가되는 후처리 효과는 전조 색상 대비를 깨지 않도록 알파를 낮춘다.
- 광역 패턴에서도 화면 전체를 덮는 불투명 VFX는 사용하지 않는다.

## 보조 축과 확장 요소

- 속성 상호작용은 1차 구현에서 `추가 피해/저항 배율 + 최소 상태 이상` 구조만 사용한다.
- 위협도는 1차 구현에서 `피해 기반 누적 + 도발 스킬` 정도만 사용하고, 세부 감쇠와 예외 규칙은 후속 확장으로 넘긴다.
- 이동 예측 보정은 제한적으로만 사용하며, 패링과 카운터 같은 핵심 전투 판정은 서버 확정 기준을 유지한다.
- 복잡한 다중 히트 패링, 강제 타게팅 패턴, 반복 제어 저항은 확장 단계에서 다룬다.

## 속성 및 상태 이상 (v1 최소 정의)

### 속성 종류와 효과

| 속성 | 추가 피해 배율 | 상태 이상 | 상태 이상 조건 |
|---|---|---|---|
| 번개 | ×1.0 (기본) | 감전 | `StatusBuildUp` 누적 100 이상 |
| 독 | ×1.0 (기본) | 중독 | `StatusBuildUp` 누적 100 이상 |
| 화염 | ×1.0 (기본) | 화상 | `StatusBuildUp` 누적 100 이상 |
| 얼음 | ×1.0 (기본) | 빙결 | `StatusBuildUp` 누적 100 이상 |

- 약점 속성 적중 시 피해 ×1.25 배율 적용.
- 저항 속성 적중 시 피해 ×0.75, 상태 이상 지속시간 ×0.3 적용.
- `StatusBuildUp` 누적은 대상별로 독립 관리하며, 상태 이상 발동 시 0으로 초기화된다.

### 상태 이상 효과 정의

| 상태 이상 | 효과 | 기본 지속시간 | 중첩 |
|---|---|---|---|
| 감전 | 이동속도 -20%, 플레이어 대상 시 추가 스턴 없음 | 2초 | 불가, 재발동 시 타이머 초기화 |
| 중독 | 매 1초마다 MaxHP의 1% DoT | 5초 | 불가, 재발동 시 타이머 초기화 |
| 화상 | 매 1초마다 MaxHP의 1.5% DoT | 3초 | 불가, 재발동 시 타이머 초기화 |
| 빙결 | 이동속도 -40% | 3초 | 불가, 재발동 시 타이머 초기화 |

- 보스 대상 상태 이상: 이동속도 효과 무효, DoT 피해와 StaggerGauge 추가 감소(+5%)만 적용.
- v1 구현에서 상태 이상 종류는 위 4종으로 제한한다. 세부 파생 규칙은 후속 확장 대상이다.

## 시점과 조작 방향

- 권장 시점은 `쿼터뷰 2.5D` 기준의 고정 또는 반고정 카메라다.
- 플레이어는 `WASD` 이동과 `마우스 방향` 조준을 기본으로 사용한다.
- 전투 재미는 조준 정밀도보다 전조 판독, 패링 타이밍, 위치 조정, 무력화 집중에 둔다.
- 기본 타게팅은 논타겟으로 두고, 근접 스킬은 빗나감 스트레스를 줄이기 위해 근거리 소프트 락 보조를 허용한다.
- 패드 대응 시 좌스틱은 이동, 우스틱은 조준 방향 역할을 맡는다.

## 권장 입력 슬롯

- `좌클릭`: 기본 공격
- `우클릭`: 패링 또는 가드
- `Space`: 회피 또는 이동기
- `Q`, `E`, `R`, `F`: 액티브 스킬 4개
- `Tab`: 아이덴티티 스킬
- `Z`, `X`: 소모품 2슬롯
- `Shift`: 보조 기동 또는 타게팅 보조

이 배치는 왼손이 이동과 핵심 스킬을 담당하고, 오른손이 조준과 패링 타이밍을 담당하도록 설계한다.

## 주요 흐름

### 기본 전투 루프

1. 플레이어는 현재 무기에 맞는 스킬 세트로 전투를 시작한다.
2. 공격 적중으로 자원과 아이덴티티 게이지를 운용한다.
3. 액티브 스킬과 아이덴티티 스킬을 조합해 딜, 방어, 유틸 역할을 수행한다.
4. 보스 전조에 맞춰 회피, 패링, 위치 조정을 반복한다.
5. 상황에 따라 즉발, 전방, 범위 지정, 돌진 계열 스킬을 섞어 사용한다.

### 보스 공략 루프

1. 파티는 보스 패턴을 확인하며 안전한 딜 타이밍을 만든다.
2. 무력화 수치가 높은 스킬과 인터럽트 타이밍을 맞춰 보스의 무력화 게이지를 깎는다.
3. 패링 또는 카운터 성공으로 즉시 빈틈을 만들고 다운 상태를 유도한다.
4. 다운 구간에 핵심 스킬과 아이덴티티 스킬을 집중해 화력을 넣는다.

### 전투 리스크

1. 사망 시 장비 내구도가 감소한다.
2. 내구도 0 장비는 핵심 능력과 아이덴티티 효과를 잃는다.
3. 체크포인트 부활 이후에도 장비 상태와 자원 관리가 다음 시도에 영향을 준다.

## 제약과 예외

- 1차 구현에서는 전투 시스템을 설명 가능한 범위로 유지하기 위해 핵심 스탯과 공식만 고정한다.
- 속성, 상태 이상, 위협도는 전투 코어를 보조하는 수준까지만 문서화하고, 복잡한 파생 규칙은 별도 확장 항목으로 관리한다.
- 무기별 아이덴티티 예시는 방향성 정의용이며 세부 수치는 별도 문서에서 다룬다.
- 데미지 계산, 무력화, 자원/쿨다운 검증은 서버 틱 루프 기준으로 처리한다.
- 클라이언트가 3D 연출을 사용하더라도 서버 전투 판정은 `XZ` 평면 기준 2D 도형 계산을 유지한다.
- 모든 스킬을 정밀 조준형으로 만들지 않고, 키 입력만으로 발동되는 즉발형과 방향 기반 스킬을 기본 비중으로 둔다.
- 무기 전환은 마을/입장 준비 구간에서만 허용하고, 전투 중 실시간 전환은 다루지 않는다.
