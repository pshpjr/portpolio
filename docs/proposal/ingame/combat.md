# 전투 시스템

## 목적

패링의 긴장감, 무기별 정체성, 파티 역할 분담을 동시에 살리는 협동 액션 전투를 정의한다.

## 핵심 규칙

- 전투는 무기 기반 클래스 구조를 따른다.
- 플레이어는 액티브 스킬 4개, 아이덴티티 스킬 1개, 소모품 2슬롯, 이동기 1개를 사용한다.
- 아이덴티티 게이지는 무기별 고유 전투 리듬을 만든다.
- 보스전의 핵심 축은 패링, 무력화, 부위 파괴, 시각적 전조 대응이다.
- 어그로와 위협도는 파티 역할 분담과 위치 선정을 유도한다.
- 속성 상호작용은 파티 시너지를 만드는 보조 축이다.
- 전투 판정은 서버 권위를 기본으로 하며, 이동만 제한적으로 예측 보정한다.

## 주요 흐름

### 기본 전투 루프

1. 플레이어는 무기에 맞는 스킬 세트로 전투를 시작한다.
2. 공격 적중으로 아이덴티티 게이지를 축적한다.
3. 액티브 스킬과 아이덴티티 스킬을 조합해 딜, 탱킹, 유틸 역할을 수행한다.
4. 보스 전조에 맞춰 회피, 패링, 위치 조정을 반복한다.

### 보스 공략 루프

1. 파티는 위협도와 포지셔닝을 관리하며 패턴을 유도한다.
2. 무력화 수치가 높은 스킬을 집중해 보스의 무력화 게이지를 깎는다.
3. 특정 패턴에서는 패링 또는 카운터에 성공해 즉시 큰 빈틈을 만든다.
4. 필요한 경우 부위 파괴로 보스 패턴을 약화하거나 봉쇄한다.

### 전투 리스크

1. 사망 시 장비 내구도가 감소한다.
2. 내구도 0 장비는 핵심 능력과 아이덴티티 효과를 잃는다.
3. 체크포인트 부활 이후에도 장비 상태와 자원 관리가 다음 시도에 영향을 준다.

## 제약과 예외

- 패링과 카운터 판정의 수치, 윈도우, 색상 규칙은 보스 문서와 함께 조정한다.
- 검방, 대검, 지팡이 같은 무기별 아이덴티티 예시는 방향성 정의용이며 세부 수치는 별도 문서에서 다룬다.
- 속성은 화염, 냉기, 번개, 독을 기본 축으로 두되 상세 공식은 구현 문서로 넘긴다.
- 데미지 계산, 상태 이상 누적, 쿨다운 검증은 서버 틱 루프 기준으로 처리한다.
