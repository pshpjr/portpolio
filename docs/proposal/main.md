# 포트폴리오 프로젝트 기획서: Dungeon of Spirits

## 1. 프로젝트 개요
- **목적**: 이직/취업용 고성능 서버 포트폴리오 (ReIW 프로젝트 문제 재설계 및 해결)
- **장르**: 소울라이크 협동 인스턴스 던전 RPG
- **핵심 컨셉**: 패링의 쾌감 + 무기 기반 올인(All-in) 빌드 + 협동 기믹 전략
- **기술 스택**: C++20 커스텀 서버, Protobuf, MySQL/SQLite, 멀티스레드(인던 싱글 스레드 루프)

## 2. 핵심 설계 결정 요약 (Final)

### 🏛 아키텍처 및 시스템
- **모듈형 모놀리스**: 단일 프로세스로 시작하되, 서비스 간 결합도를 낮춰 미래의 **인던 서버 물리 분리**가 용이한 구조.
- **인던 싱글 스레드**: 각 던전 인스턴스는 **독립적인 단일 스레드**에서 루프를 수행하여 데이터 경합 제거 및 확장성 확보.
- **계정 및 공유 창고**: 멀티 계정/캐릭터 지원 및 캐릭터 간 자원 공유를 위한 **계정 공유 창고** 시스템.

### ⚔️ 전투 및 아이템
- **무기 기반 정체성**: 무기 타입이 곧 클래스를 결정하며, 장착 중인 무기에 맞춘 **스마트 드랍 보정** 제공.
- **패링 및 속성**: 200ms 정밀 판정 및 아이템 옵션 연동. 속성 전이를 통한 협동 시너지 창출.
- **아이템 경제**: **단순 거래 횟수 제한(Trade Count)**을 통한 인플레이션 방지 및 아이템 가치 보존.

### 🏰 던전 및 보상
- **기믹 스케일링**: 파티 인원(1~4인)에 따라 협동 기믹의 개수가 동적으로 조절됨.
- **실패 페너티**: 기믹 실패 시 **경험치/드랍이 없는 정예 몬스터**가 추가 소환되어 공략 시간을 강제 지연시킴.
- **보상 이원화**: 일일/주간 제한이 있는 **고등급 미션 보상**과 무제한 파밍이 가능한 **일반 보상** 모드 운영.

## 📅 프로젝트 마일스톤

> 각 Phase의 완료 기준은 해당 상태까지 실제로 동작하는 체크포인트를 기준으로 한다.
>
> **기간 정책**: 월별 가용 시간(평일 1h × 20일 + 주말 4.5h × 8일 ≈ 56h/월) 기준으로 Phase당 1개월을 기본 단위로 잡되, 실제 진행 속도를 측정하며 조정한다. Phase 1-A처럼 XL 항목이 몰린 구간이나 Phase 4처럼 볼륨이 큰 구간은 2개월로 늘릴 수 있다.

---

### Phase 1-A: 서버 코어 (1개월차)

**체크포인트**: 클라이언트가 서버에 접속하고 패킷을 주고받는다.

| 항목 | 규모 |
|------|------|
| TCP 서버 + Asio I/O 스레드풀 | XL |
| Protobuf 패킷 정의 + 직렬화/역직렬화 | XL |
| 글로벌 워커 스레드 + Job Queue + Dispatcher | XL |
| DB 비동기 레이어 (MySQL/SQLite, Write-back) | L |
| JSON 정적 테이블 로드 (Item/Skill/Monster/OptionTable) | M |

---

### Phase 1-B: 계정 / 마을 (2개월차)

**체크포인트**: 계정을 만들고 캐릭터를 선택한 뒤 마을에서 채팅하며 파티를 꾸릴 수 있다.

| 항목 | 규모 |
|------|------|
| 계정 생성 / 로그인 / 세션 관리 | S |
| 캐릭터 생성 / 선택 (멀티 캐릭터) | S |
| 인벤토리 CRUD + 아이템 귀속 타입 | M |
| 마을 입장 + 위치 동기화 (브로드캐스트) | M |
| 채팅 시스템 (전체 / 파티) | S |
| 파티 시스템 (생성·초대·추방·해산·파티 찾기) | M |

---

### Phase 2: 인던 코어 + 기본 전투 (3개월차)

**체크포인트**: 파티가 던전에 입장해 일반 몬스터를 기본 공격으로 잡고, 사망·부활할 수 있다. 인던 루프가 30fps로 안정 동작한다.

| 항목 | 규모 |
|------|------|
| 던전 입장 요청 / 동의 / 인스턴스 생성 + 모드 선택 | M |
| 인던 싱글 스레드 루프 (30fps, 엔티티 관리) | L |
| 이동 / 위치 동기화 (CSP) | M |
| 기본 공격 / 피해 판정 (서버 권위) | M |
| 사망 / 부활 시스템 | M |
| 일반 몬스터 AI (어그로·추적·공격 FSM) | M |
| 구역 관문 개폐 (몬스터 전멸 트리거) | S |

---

### Phase 3: 전투 심화 + 스킬 + 중간보스 (4개월차)

**체크포인트**: 스킬을 사용하고, 패링으로 보스 시전을 캔슬하며, 무력화 게이지를 소진해 Staggered 상태를 만들 수 있다.

| 항목 | 규모 |
|------|------|
| 스킬 시스템 (무기별 고정 세트, 4슬롯 구성, 포인트 투자) | L |
| Stagger Power 등급 스킬 데이터 필드 적용 | M |
| 패링 (200ms 타이밍 윈도우 서버 판정) | M |
| 무력화 시스템 (Stagger Gauge, Staggered 상태, 누적 저항) | M |
| 캔슬 시스템 (시전 바 색상별 분기, Heavy/Crush 조건, 실패 페널티) | M |
| 중간 보스 AI (슈퍼아머, 그로기 게이지, HP 트리거 소환) | L |
| 소환 그룹 시스템 (SummonGroup, 소환자 사망 시 일괄 처리) | M |

---

### Phase 4: 보스 + 기믹 + 보상 (5개월차)

**체크포인트**: 주술사를 Phase1~2 완전 공략할 수 있다. 기믹 실패 시 페널티 소환이 뜨고, 클리어 후 랜덤 옵션 아이템이 드랍된다.

| 항목 | 규모 |
|------|------|
| 보스(주술사) AI - Phase1 패턴 (즉발/노란/빨간 시전 바) | XL |
| 보스(주술사) AI - Phase2 패턴 + 광폭화 (8분 타이머) | XL |
| 기믹 (협동 발판/레버 + 인원수 스케일링) | L |
| 기믹 실패 페널티 소환 (중복 소환 제한 포함) | M |
| 랜덤 옵션 풀 드랍 (OptionTable 기반 롤링) | M |
| 드랍 처리 (개인 드랍, 자동 줍기, 골드) | M |
| 보스 보물상자 + 개인 보상 UI | S |
| 보상 이원화 (제한형 / 무제한형 모드 분기) | S |
| 내구도 시스템 + NPC 상점 (구매·판매·수리) | S+S |
| 계정 공유 창고 | M |

---

### Phase 5: 런타임 + 성능 검증 (6개월차)

**체크포인트**: 더미 클라이언트 200대 기준 전 인던이 30fps를 유지하고 성능 리포트가 출력된다.

| 항목 | 규모 |
|------|------|
| 런타임 핫 리로드 (이중 버퍼 + 관리자 콘솔 트리거) | M |
| 더미 클라이언트 (마을 100명 + 인던 100명, 3종 시나리오) | L |
| 서버 메트릭 수집 (TPS, Tick Time, 메모리, DB I/O) | M |
| 성능 리포트 CSV → 마크다운 변환 + 임계점 분석 | M |
| 전체 버그 수정 + 엣지 케이스 처리 | L |

---

### 리스크 관리

#### 절대 빠져선 안 되는 항목 (Must-Have)
포트폴리오의 기술 증명 가치와 직결되는 항목. 이 중 하나라도 없으면 시연 불가.

- TCP/Protobuf 통신 프레임워크
- 멀티스레드 아키텍처 (3종 스레드풀, 인던 싱글 스레드)
- DB 비동기 레이어 + 인메모리 캐싱
- 인던 싱글 스레드 루프 (30fps)
- 무력화 + 캔슬 시스템 (확정 범위)
- 보스(주술사) AI — 최소 Phase1 + 캔슬 가능 패턴 1종 이상
- 랜덤 옵션 풀 드랍 (OptionTable 롤링)
- 더미 클라이언트 + 성능 테스트 리포트

#### 일정이 밀릴 경우 잘라낼 수 있는 항목 (Deferrable)

| 항목 | 대안 |
|------|------|
| 보스 Phase2 패턴 전체 | Phase1 강화형으로 단순화. Phase 전환 자체는 구현 |
| 기믹 (협동 발판/레버) | "전원 구역 진입 시 문 열림" 단순 트리거로 대체. 스케일링 로직만 유지 |
| 기믹 실패 페널티 소환 | 소환 없이 N초 진행 잠금 타임 페널티로 대체 |
| 계정 공유 창고 | 인벤토리만 완성. 설계는 architecture.md에 명시 유지 |
| 런타임 핫 리로드 | 서버 재시작으로 대체. 코드 구조만 교체 가능하게 설계 |
| 보상 이원화 | 단일 보상 모드로 통합 |
| 파티 찾기 (공개 파티 목록) | 초대 기반 파티 구성만 유지 |
| 성능 리포트 자동화 | CSV만 생성 후 수동 정리 |

## 📅 Phase 6: Vision (Post-Portfolio) - "확장성 증명"
- **분산 서버**: 마을 서버와 던전 서버의 물리적 분리 및 Proxy 서버 도입.
- **절차적 던전 생성 (PCG)**: 방(Room) 단위의 타일셋(Tileset)을 무작위로 조합하여 매번 구조가 변하는 무한 던전 시스템 도입.
- **글로벌 서비스**: Redis를 활용한 글로벌 세션 및 전 서버 랭킹 시스템 구축.
