# 데이터 드리븐 및 런타임 핫 리로드

## 1. 개요
본 서버는 게임 밸런스 조정(아이템 옵션, 스킬 계수, 몬스터 스탯 등)을 서버 재시작 없이 실시간으로 반영하기 위해 **데이터 드리븐(Data-Driven)** 설계와 **런타임 핫 리로드(Hot-Reload)**를 지원한다.

## 2. 정적 데이터 관리 (JSON)
모든 정적 데이터는 JSON 형식으로 관리되며, 서버 시작 시 전량 메모리에 로드된다.
- **주요 테이블**:
  - `ItemTable`: 기본 아이템 정보 및 등급별 옵션 개수 정의.
  - **`OptionTable`**: `OptionID`별 능력치 종류, 수치 범위(Min/Max), 적용 공식을 정의.
  - `SkillTable`: 무기 타입별 스킬트리 구조 및 레벨별 계수 정의.
  - `MonsterTable`: 몬스터 스탯 및 기믹 실패 시 소환될 정예 몬스터 정보.

## 3. 아이템 랜덤 옵션 풀 (Option Pool)
디아블로식 랜덤 옵션 생성을 위해 `OptionID` 기반의 풀(Pool) 시스템을 운영한다.
- **옵션 생성 규칙**: 아이템 드랍 시 해당 아이템 등급에 맞는 `OptionID` 풀에서 무작위로 추출하고, `OptionTable`에 정의된 범위 내에서 수치를 롤링한다.
- **네트워크 최적화**: 생성된 옵션은 `OptionID(uint16) + Value(int32)` 형태로 패킷에 담겨 클라이언트에 전달된다.

## 4. 런타임 핫 리로드 (Hot-Reload)
서버 운영 중 관리자 명령을 통해 특정 데이터를 실시간으로 교체할 수 있다.
- **이중 버퍼(Double Buffering)**: 새로운 데이터를 별도 메모리 영역에 로드한 후, 검증이 완료되면 포인터를 교체하는 방식으로 안전성을 확보한다.
- **인스턴스 일관성**:
  - **기존 인스턴스**: 이미 생성된 던전 인스턴스는 생성 시점의 데이터 버전을 계속 유지한다.
  - **신규 인스턴스**: 리로드 완료 후 생성되는 인스턴스부터 새로운 데이터를 참조한다.
- **트리거**: 관리자 콘솔 커맨드(`reload all` 또는 `reload item`)를 통해 실행한다.
