# 데이터 드리븐 및 런타임 핫 리로드

## 목적

밸런스 조정과 콘텐츠 데이터 변경을 서버 재시작 없이 반영할 수 있는 운영 방향을 정의한다.

## 핵심 규칙

- 정적 게임 데이터는 파일 기반 데이터 테이블로 관리한다.
- 서버 시작 시 정적 데이터를 메모리에 적재한다.
- 아이템 옵션, 스킬 계수, 몬스터 수치는 데이터 중심으로 다룬다.
- 운영 중 일부 데이터는 핫 리로드로 교체할 수 있다.
- 기존 인스턴스와 신규 인스턴스는 데이터 버전 일관성을 분리해서 유지한다.

## 주요 흐름

### 데이터 적재

1. 서버 시작 시 아이템, 옵션, 스킬, 몬스터 같은 정적 데이터를 읽는다.
2. 런타임은 메모리 상의 테이블을 기준으로 참조한다.

### 핫 리로드

1. 운영자가 특정 데이터 리로드 명령을 실행한다.
2. 서버는 새 데이터를 별도 버퍼에 적재하고 검증한다.
3. 검증이 끝나면 참조 포인터를 교체한다.
4. 기존 인스턴스는 이전 버전을 유지하고, 이후 생성 인스턴스부터 새 버전을 사용한다.

## 제약과 예외

- 데이터 포맷, 명령어 문자열, 버퍼 구조 같은 구현 세부는 서버 구현 문서에서 확정한다.
- 핫 리로드 대상은 정적 밸런스 데이터 중심으로 한정한다.
- 인스턴스별 버전 유지 정책은 안정성을 우선한다.
