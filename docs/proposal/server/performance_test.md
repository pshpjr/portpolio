# 성능 테스트 및 더미 클라이언트

## 1. 개요
본 프로젝트는 **200명 이상의 동시 접속** 환경에서 서버의 안정성과 성능을 검증하기 위해 **더미 클라이언트(Dummy Client)**를 통한 스트레스 테스트를 수행한다.

## 2. 테스트 목표 및 규모
- **동시 접속**: 최대 200명 (마을 100명 + 인던 100명).
- **인던 부하**: 최대 25~50개의 인스턴스가 동시에 가동되는 상황 (인원수에 따라 유동적).
- **기술 검증**: 인던 싱글 스레드 모델의 틱(Tick) 처리 시간 및 CPU/메모리 점유율 분석.

## 3. 더미 클라이언트 시나리오
더미 클라이언트는 서버와 동일한 **Protobuf 패킷**을 사용하며, 다음 시나리오를 자동으로 반복한다.
- **마을 스트레스**: 100명이 1초마다 무작위 위치 이동 및 채팅 패킷 전송.
- **던전 입장/클리어**: 25개 파티가 동시 입장을 시도하고, 인던 내에서 스킬 사용 및 몬스터 전투(무작위 공격) 수행.
- **기믹 실패 유발**: 일부 더미 파티는 고의로 기믹 해결을 지연시켜 **정예 몬스터 소환 페널티** 상황의 부하를 유도한다.

## 4. 서버 메트릭 수집 및 분석
테스트 수행 중 서버는 다음 데이터를 실시간으로 수집한다.
- **패킷 처리량 (TPS)**: 초당 처리되는 패킷의 수.
- **인던 처리 지연 (Tick Time)**: 30 FPS 기준, 1틱(약 33ms) 내에 모든 로직이 완료되는지 검증.
- **메모리 점유율**: 동접 증가 및 인던 생성 시 메모리 누수 여부 확인.
- **DB 비동기 처리**: 경매장이나 보상 지급 시 발생하는 DB I/O의 응답 지연 시간.

## 5. 성능 리포트 작성
테스트 결과는 CSV 형식으로 저장되며, 최종적으로 마크다운 형식의 **성능 검증 리포트**로 변환되어 포트폴리오의 기술적 근거로 활용된다.
- **최종 검증**: 200명 동접 상황에서도 모든 인던 인스턴스가 안정적인 30 FPS를 유지하는지 확인한다.
- **임계점 분석**: 서버가 허용 가능한 최대 동접 및 인던 수를 측정하여 확장성 지표로 제시한다.
