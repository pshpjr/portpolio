# 성능 테스트 및 더미 클라이언트

## 목적

동시 접속과 인던 부하 상황에서 서버 구조가 목표 성능을 유지하는지 검증하는 기준을 정의한다.

## 핵심 규칙

- 목표 검증 환경은 동시 접속 200명 이상이다.
- 마을 부하와 인던 부하를 동시에 측정한다.
- 더미 클라이언트는 실제 서버 패킷 규약과 최대한 같은 형태로 동작한다.
- 성능 평가는 TPS, 틱 지연, 메모리, DB I/O를 함께 본다.
- 결과는 최종적으로 포트폴리오용 성능 리포트로 정리한다.

## 주요 흐름

### 부하 시나리오

1. 마을 더미 클라이언트가 이동과 채팅 부하를 반복한다.
2. 인던 더미 파티가 입장, 전투, 스킬 사용을 반복한다.
3. 일부 시나리오는 기믹 실패를 유도해 페널티 몬스터 부하도 포함한다.

### 메트릭 수집

1. 서버가 패킷 처리량과 틱 시간을 기록한다.
2. 메모리 사용량과 누수 여부를 확인한다.
3. DB 비동기 작업의 지연 시간을 측정한다.

### 결과 정리

1. 원시 결과를 수집한다.
2. 임계 지점과 병목을 분석한다.
3. 포트폴리오 제출용 성능 검증 문서로 정리한다.

## 제약과 예외

- 더미 클라이언트 구현 방식과 리포트 포맷은 구현 문서에서 확정한다.
- 동접 수와 인던 수 같은 목표 수치는 기획 기준이며 환경에 따라 조정될 수 있다.
- 최종 합격 기준은 모든 인던이 안정적으로 30 FPS 수준을 유지하는 것이다.
