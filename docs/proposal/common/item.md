# 아이템 및 장비 시스템

## 목적

아이템 파밍, 장비 성장, 거래 제한을 통해 무기 기반 올인 빌드를 밀어주면서도 "무엇을 주웠고 왜 강해졌는지"가 설명 가능한 장비 구조를 만든다.

## 핵심 규칙

- 모든 전리품은 개인 드롭을 기본으로 한다.
- 장착 무기 타입 기준 스마트 드롭은 `확정 맞춤형`이 아니라 `soft bias`로 사용한다.
- 아이템은 `무기`, `방어구`, `장신구`, `스킬북`, `소모품`, `재련/강화 재료`로 나눈다.
- 장비 등급은 `노멀 -> 언커먼 -> 레어 -> 에픽` 순으로 가치가 올라간다.
- 전투 스탯은 `레벨 기본값 + 무기 기본값 + 장비 고정 스탯 + 랜덤 옵션 + 임시 버프` 순서로 조립한다.
- 재련은 한 번에 한 옵션만 다시 굴리는 방식으로 제한한다.
- 재련을 시도한 아이템은 계정 귀속이 된다.
- 거래 가능 횟수가 0이 되면 타인 거래는 막히지만 계정 공유 창고 이동은 허용한다.
- 사망 시 장착 무기의 내구도가 감소하고, 내구도 0인 무기는 옵션과 전투 성능이 비활성화된다.

## v1 장비 구조

### 장비 슬롯

- `Weapon` 1칸: 무기 타입, 아이덴티티, 자원 구조, 기본 전투 리듬을 결정한다.
- `Armor` 4칸: `머리`, `상의`, `장갑`, `신발`로 나누고 생존/기동 보정을 담당한다.
- `Accessory` 2칸: `목걸이`, `반지`로 두고 치명, 쿨다운, 패링 보정 같은 보조 축을 담당한다.

이 구조는 "무기 1개로 정체성을 잡고, 나머지 슬롯은 그 무기를 보완한다"는 방향을 유지하기 위한 v1 고정안이다.

### 아이템 계층

- `무기`: 가장 큰 전투 성능 변화와 아이덴티티 차이를 담당한다.
- `방어구`: `MaxHP`, `Defense`, `DamageReduction`, `MoveSpeedBonus` 같은 생존/안정성 값을 제공한다.
- `장신구`: `CriticalChance`, `CriticalDamage`, `CooldownReduction`, `ParryWindowBonus`, `AttackSpeed`, `CastSpeed` 같은 보조 전투축을 제공한다.
- `스킬북`: 스킬 습득과 스킬 레벨 확장에 사용한다.
- `소모품`: 회복, 버프, 유틸성 기능을 담당한다.
- `재료`: 재련, 수리, 후속 확장 강화 시스템의 비용 재화다.

## 스탯 출처와 옵션 규칙

### 1. 고정 스탯 출처

1. 레벨 테이블이 플레이어 공통 기본값을 준다.
2. 무기 테이블이 무기 기본 전투값과 자원/아이덴티티 구조를 준다.
3. 방어구/장신구 템플릿이 슬롯별 고정 스탯을 준다.
4. 랜덤 옵션이 특정 빌드 축을 미세 조정한다.
5. 버프, 스킬 변형, 임시 효과가 전투 중 최종 보정을 준다.

### 2. 장비군별 옵션 성격

- `무기 옵션`: `WeaponPower`, `CriticalChance`, `CriticalDamage`, `StaggerPower`, `ElementAttack_X`, `IdentityGaugeGain` 같은 공격/무력화 축 중심
- `방어구 옵션`: `MaxHP`, `Defense`, `DamageReduction`, `MoveSpeedBonus` 같은 안정성 축 중심
- `장신구 옵션`: `CooldownReduction`, `ParryWindowBonus`, `AttackSpeed`, `CastSpeed`, `ThreatGen` 같은 운용 보조 축 중심

같은 옵션이라도 모든 장비군에 열지 않고, 슬롯별 역할이 읽히도록 옵션 풀을 분리한다.

### 3. 스마트 드롭 규칙

- 장착 무기 타입과 같은 계열 장비/스킬북 드롭 확률을 우선 보정한다.
- 다만 전체 드롭을 한 무기 타입으로 잠그지 않고, 오프빌드 장비와 공용 재료 드롭 여지는 유지한다.
- v1 기준으로는 "현재 빌드를 밀어주되 미래 전환 가능성은 남긴다"가 원칙이다.

## 주요 흐름

### 파밍과 장착

1. 던전 보상으로 개인 드롭 아이템을 획득한다.
2. 장착 무기 타입에 맞는 무기, 장신구, 스킬북이 우선 보정된다.
3. 플레이어는 등급, 고정 스탯, 랜덤 옵션을 비교해 기존 장비와 교체 여부를 판단한다.
4. 장비 교체 후 최종 전투 스탯이 즉시 재계산된다.

### 재련과 귀속

1. 플레이어는 원하는 옵션 하나를 선택해 재련한다.
2. 재련 시도 횟수가 늘수록 비용이 빠르게 증가한다.
3. 재련을 시작한 장비는 계정 귀속으로 전환된다.
4. 귀속 이후에도 같은 계정 내 캐릭터 간 이동은 공유 창고로 허용한다.

### 수리와 유지

1. 캐릭터 사망 시 장착 무기 내구도가 즉시 감소한다.
2. 내구도 0 무기는 파괴되지 않지만 핵심 성능과 옵션이 비활성화된다.
3. 마을에서 골드를 지불해 내구도를 복구한다.

### 거래와 전환

1. 아이템은 정해진 거래 가능 횟수를 가진다.
2. 유저 간 거래가 성사될 때마다 횟수가 차감된다.
3. 거래 가능 횟수가 0이 되면 계정 귀속 상태가 된다.
4. 고정 스탯만 보고 쓰는 저가 장비와 옵션 가치가 높은 고급 장비가 서로 다른 거래 동기를 만든다.

## v1 경계

- 방어구와 장신구는 무기처럼 복잡한 강화 트리를 가지지 않는다.
- 강화/초월/세트 효과 같은 다층 성장 시스템은 v1 범위 밖으로 둔다.
- 재련은 "옵션 1개 재굴림"까지만 허용하고, 옵션 잠금/복수 옵션 동시 교체는 확장으로 넘긴다.
- 룬, 보석, 각인처럼 별도 장비판을 추가하는 시스템은 도입하지 않는다.
- `OptionID + Value`, 소유 인스턴스 구조, 실제 DB 정규화는 구현 문서에서 확정한다.
