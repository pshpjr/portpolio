# 아이템 및 장비 시스템

## 1. 개요
본 프로젝트의 아이템 시스템은 **디아블로식 루트(Loot) 시스템**과 **전략적 자원 관리**를 지향한다. 모든 아이템은 개인별 독립 드랍(Individual Drop)을 원칙으로 하며, 유저가 선택한 무기에 올인(All-in)할 수 있도록 설계되었다.

## 2. 아이템 등급 및 가치 (Grades)
아이템은 희귀도에 따라 4단계로 구분되며, 등급이 높을수록 기본 성능과 랜덤 옵션의 개수가 증가한다.
- **노멀 (Normal)**: 기본 성능만 보유.
- **언커먼 (Uncommon)**: 1~2개의 보조 옵션 부여.
- **레어 (Rare)**: 2~3개의 강력한 옵션 부여 및 옵션 재련 가능.
- **에픽 (Epic)**: 4~5개의 옵션과 무기 고유의 특수 효과 부여 가능.

## 3. 랜덤 옵션 시스템 (OptionID + Value)
모든 무기는 획득 시 정해진 풀(Pool) 내에서 무작위 옵션이 부여된다.
- **데이터 구조**: 네트워크 전송 최적화를 위해 `OptionID(uint16) + Value(int32)` 형태로 관리된다.
- **옵션 종류**: 공격력, 치명타, 속성 강화, **무력화 수치 증가**, **부위 파괴 수치 증가**, 패링 윈도우 확장 등.
- **스마트 드랍**: 던전 공략 시 유저가 **현재 장착 중인 무기 타입**의 아이템 및 스킬북이 드랍될 확률이 50% 이상 보정된다.

## 4. 장비 내구도 및 수리 (Durability)
- **내구도 소모**: 캐릭터 사망 시 장착한 모든 무기의 내구도가 최대치의 10%만큼 즉시 감소한다.
- **손상 상태 (Durability 0)**: 내구도가 0이 된 무기는 파괴되지 않으나, **모든 공격력 보너스와 옵션 효과가 비활성화**된다. (착용하지 않은 것과 동일한 성능)
- **수리**: 마을 대장장이 NPC를 통해 골드를 지불하고 내구도를 복구할 수 있다. (핵심 Gold Sink)

## 5. 아이템 옵션 재련 (Enchanting / Reroll)
디아블로식 재련 시스템을 통해 최적의 옵션을 맞출 수 있다.
- **재련 규칙**: 여러 옵션 중 **단 하나**만 선택하여 다시 굴릴 수 있으며, 선택된 슬롯 외의 다른 슬롯은 영구 잠금된다.
- **비용**: 동일 아이템을 반복 재련할수록 비용이 기하급수적으로 상승한다.
- **귀속**: 재련을 시도한 아이템은 즉시 **'계정 귀속'** 상태가 되어 타인과 거래할 수 없다.

## 6. 귀속 및 거래 제한 정책 (Binding & Trade)
- **단순 거래 횟수 제한**: 아이템마다 정해진 거래 횟수(예: 3회)가 존재하며, 거래 시마다 1회씩 차감된다.
- **귀속 전환**: 거래 횟수가 0이 되면 **'계정 귀속'** 상태가 된다.
- **계정 공유 창고**: 거래 횟수가 0이 되어도 계정 내 다른 캐릭터에게는 **공유 창고**를 통해 자유롭게 아이템을 넘겨줄 수 있다.

## 7. 소모품 사용 규칙 (Quickslots)
- **슬롯 구성**: 퀵바에 2개의 소모품 슬롯(단축키 1, 2)이 존재한다.
- **수량 제한**: 각 슬롯에는 **최대 5개**까지만 아이템을 장착할 수 있으며, 던전 내에서는 슬롯 구성을 변경할 수 없다.
- **독립 쿨다운**: 1번 슬롯과 2번 슬롯은 서로 쿨다운이 겹치지 않고 독립적으로 동작한다.

## 8. 데이터 테이블 구조 (Data Table)

### Item 테이블 (정적 데이터)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| item_id | uint32 | 고유 ID (PK) |
| name | string | 아이템 이름 |
| grade | enum | NORMAL, UNCOMMON, RARE, EPIC |
| type | enum | WEAPON, CONSUMABLE, SKILLBOOK, ETC |
| sub_type | int | 무기 타입(검방, 지팡이 등) 또는 소비템 종류 |
| base_atk | int | 기본 공격력 |
| max_durability | int | 최대 내구도 |
| trade_count_limit | int | 초기 거래 가능 횟수 |

### OptionTable (정적 데이터)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| option_id | uint16 | 옵션 고유 ID (PK) |
| name | string | 옵션 이름 (예: 추가 화염 피해) |
| value_min | int32 | 해당 옵션의 최소 수치 |
| value_max | int32 | 해당 옵션의 최대 수치 |
| calc_type | enum | FLAT, PERCENT, MS |

### UserInventory (동적 데이터)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| instance_id | uint64 | 아이템 인스턴스 고유 ID (PK) |
| user_id | uint64 | 소유 유저 ID |
| item_id | uint32 | 기본 아이템 정보 참조 |
| current_durability | int | 현재 내구도 |
| remaining_trade_count| int | 남은 거래 가능 횟수 |
| is_account_bound | bool | 계정 귀속 여부 (재련 시 true) |
| options_json | string | 적용된 OptionID + Value 목록 (JSON 또는 Bitmask) |
| rerolled_option_id | uint16 | 재련된 옵션의 ID (0이면 재련 이력 없음) |
| reroll_count | int | 재련 시도 횟수 (비용 계산용) |
