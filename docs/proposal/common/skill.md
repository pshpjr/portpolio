# 스킬 및 성장 시스템

## 목적

무기 기반 정체성을 유지하면서도 적은 슬롯 안에서 빌드 다양성과 성장 선택을 제공한다.

## 핵심 규칙

- 스킬 구조는 무기 기반 클래스 개념을 따른다.
- 각 무기는 고유 아이덴티티 게이지와 아이덴티티 스킬을 가진다.
- 액티브 스킬은 적은 수를 유지하되, 성장과 변형 선택으로 빌드 차이를 만든다.
- 스킬 습득은 기본적으로 던전 드롭 기반이다.
- 장착 무기 타입에 맞는 스킬북과 장비 드롭을 우선 보정한다.
- 던전 내부에서는 스킬 슬롯과 주요 구성을 바꾸지 않는다.

## 주요 흐름

### 스킬 습득과 성장

1. 플레이어는 던전에서 스킬북을 획득해 새 스킬을 배운다.
2. 레벨업으로 얻은 포인트를 스킬 레벨과 변형 선택에 투자한다.
3. 룬이나 추가 효과를 스킬 슬롯에 장착해 운용 방식을 세분화한다.
4. 장착 무기 중심으로 파밍을 이어가며 빌드를 고도화한다.

### 아이덴티티 운용

1. 전투 중 공격 적중으로 무기별 게이지를 모은다.
2. 게이지를 소모해 무기 고유의 강한 순간 성능을 발동한다.
3. 검방은 보호와 탱킹, 지팡이는 폭발적 제어, 대검은 공격 집중 같은 역할 차이를 만든다.

## 제약과 예외

- 아이덴티티 예시는 방향성 정의용이며 수치와 세부 밸런스는 후속 조정 대상이다.
- 트라이포드식 변형은 빌드 다양성 확보가 목적이며 슬롯 수 자체는 크게 늘리지 않는다.
- 로그인 시 여러 무기 타입의 스킬 정보를 메모리에 올리는 방향을 전제로 한다.
- 쿨다운, 자원 검증, 동기화 방식은 구현 문서에서 확정한다.
