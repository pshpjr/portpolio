# 스킬 및 성장 시스템

## 목적

무기 기반 정체성을 유지하면서도 적은 슬롯 안에서 빌드 다양성과 성장 선택을 제공한다.

## 핵심 규칙

- 스킬 구조는 무기 기반 클래스 개념을 따른다.
- 각 무기는 고유 아이덴티티 게이지와 아이덴티티 스킬을 가진다.
- 액티브 스킬은 적은 수를 유지하되, 성장과 변형 선택으로 빌드 차이를 만든다.
- 스킬 습득은 기본적으로 던전 드롭 기반이다.
- 장착 무기 타입에 맞는 스킬북과 장비 드롭을 우선 보정한다.
- 던전 내부에서는 스킬 슬롯과 주요 구성을 바꾸지 않는다.
- 기본 조작은 `WASD` 이동, `마우스 방향` 조준, `Q/E/R/F` 액티브 4슬롯을 기준으로 잡는다.

## 스킬 사용 방식

- 즉발 자기중심 스킬은 키 입력만으로 즉시 발동한다.
- 전방 공격 스킬은 현재 `마우스 방향`을 기준으로 부채꼴 또는 직사각형 판정을 생성한다.
- 돌진/추격 스킬은 현재 입력 방향 또는 마우스 방향을 사용한다.
- 범위 지정 스킬은 키 입력 후 짧은 조준 표시를 띄우고, 확인 입력으로 확정한다.
- 패링/카운터 계열은 짧은 유효 시간 안에 서버 확정 판정으로 성공 여부를 판단한다.
- 서버는 모든 스킬을 `원형`, `부채꼴`, `직사각형`, `이동 경로`, `투사체 라인` 중 하나로 환원해 처리한다.

## 권장 입력 예시

- `좌클릭`: 기본 공격
- `우클릭`: 패링 또는 가드
- `Q`, `E`, `R`, `F`: 액티브 스킬 4개
- `Space`: 회피 또는 이동기
- `Tab`: 아이덴티티 스킬
- `Z`, `X`: 소모품

숫자키 중심 배치는 `WASD`와 손이 멀어지므로 1차 기본안에서는 우선순위를 낮춘다.

## 주요 흐름

### 스킬 습득과 성장

1. 플레이어는 던전에서 스킬북을 획득해 새 스킬을 배운다.
2. 레벨업으로 얻은 포인트를 스킬 레벨과 변형 선택에 투자한다.
3. 룬이나 추가 효과를 스킬 슬롯에 장착해 운용 방식을 세분화한다.
4. 장착 무기 중심으로 파밍을 이어가며 빌드를 고도화한다.

### 아이덴티티 운용

1. 전투 중 공격 적중으로 무기별 게이지를 모은다.
2. 게이지를 소모해 무기 고유의 강한 순간 성능을 발동한다.
3. 검방은 보호와 탱킹, 지팡이는 폭발적 제어, 대검은 공격 집중 같은 역할 차이를 만든다.

## 무기별 조작 예시

### 검방

- `좌클릭`: 짧은 3타 기본 연계
- `우클릭`: 패링 또는 가드 유지
- `Q`: 전방 방패 강타, 무력화 기여가 높은 근접 스킬
- `E`: 짧은 돌진 후 베기, 진입과 위치 보정용
- `R`: 도발 또는 보호막 부여, 파티 보호 유틸
- `F`: `Heavy` 등급 방패 내려치기, 캔슬 대응용
- `Tab`: 아이덴티티 발동, 일정 시간 방어 강화와 패링 보상 증가

검방은 보스 정면에서 패링 타이밍과 캔슬 대응을 담당하는 조작 리듬을 가진다.

### 대검

- `좌클릭`: 느리지만 범위가 넓은 기본 연계
- `우클릭`: 카운터 자세 또는 차지 시작
- `Q`: 넓은 전방 베기, 잡몹 정리용
- `E`: 전진 참격, 추격 또는 약한 이동기
- `R`: 차지 강공격, 무력화 게이지 압박용
- `F`: 짧은 슈퍼아머를 동반한 내려찍기
- `Tab`: 아이덴티티 발동, 일정 시간 고위력 일점 폭딜 창출

대검은 패링보다 다운 타이밍과 무력화 구간 폭딜 집중에 더 큰 비중을 둔다.

### 지팡이

- `좌클릭`: 자동 유도 성향의 약한 원거리 평타
- `우클릭`: 짧은 시전 집중 또는 긴급 보호막
- `Q`: 직선 투사체 또는 관통 마법
- `E`: 지정 위치 폭발, 장판 제어용
- `R`: 속성 약점 대응 디버프 또는 구속
- `F`: 긴 시전의 고위력 주문, 파티가 만든 빈틈에 사용
- `Tab`: 아이덴티티 발동, 광역 제어 또는 시전 강화 상태 진입

지팡이는 안전 거리 유지, 약점 속성 활용, 다운 타임 화력 집중을 담당한다.

## 제약과 예외

- 아이덴티티 예시는 방향성 정의용이며 수치와 세부 밸런스는 후속 조정 대상이다.
- 트라이포드식 변형은 빌드 다양성 확보가 목적이며 슬롯 수 자체는 크게 늘리지 않는다.
- 로그인 시 여러 무기 타입의 스킬 정보를 메모리에 올리는 방향을 전제로 한다.
- 쿨다운, 자원 검증, 동기화 방식은 구현 문서에서 확정한다.
- 범위 지정 스킬은 전체 스킬 중 소수만 사용하고, 과도한 클릭 조작이 반복되지 않게 한다.
