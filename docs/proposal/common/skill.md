# 스킬 및 성장 시스템

## 목적

무기 전환이 가능한 구조 안에서도 무기별 손맛과 성장 선택을 유지하는 스킬 규칙을 정의한다.

## 핵심 규칙

- 스킬 구조는 무기 기반 클래스 개념을 따른다.
- 각 무기는 고유 아이덴티티 게이지와 아이덴티티 스킬을 가진다.
- 액티브 스킬은 `Q/E/R/F` 4개로 고정하고, 성장과 변형 선택으로 빌드 차이를 만든다.
- 캐릭터는 마을에서 무기를 바꿀 수 있지만, 던전 내부에서는 현재 무기와 스킬 구성을 바꾸지 않는다.
- 무기마다 별도 스킬 프리셋을 저장한다.
- 스킬 습득은 기본적으로 던전 드롭과 마을 훈련 해금 기반이다.
- 전리품 선호 무기에 맞는 스킬북 드롭을 우선 보정하되, 오프빌드 드롭 여지는 남긴다.
- 기본 조작은 `WASD` 이동, `마우스 방향` 조준, `Q/E/R/F` 액티브 4슬롯을 기준으로 잡는다.

## 무기 전환 규칙

- 무기 전환은 마을, 훈련장, 던전 입장 준비 구간에서만 가능하다.
- 던전 입장 시 현재 무기, 스킬 프리셋, 전리품 선호 무기를 함께 잠근다.
- 다른 무기를 새로 장착하면 해당 무기의 기본 스킬 4종이 자동 연결되고, 이후에는 무기별 프리셋을 개별 성장시킨다.
- 전투 중 즉시 무기 교체는 v1 범위 밖이다.

## 스킬 사용 방식

- 즉발 자기중심 스킬은 키 입력만으로 즉시 발동한다.
- 전방 공격 스킬은 현재 `마우스 방향`을 기준으로 부채꼴 또는 직사각형 판정을 생성한다.
- 돌진/추격 스킬은 현재 입력 방향 또는 마우스 방향을 사용한다.
- 범위 지정 스킬은 키 입력 후 짧은 조준 표시를 띄우고, 확인 입력으로 확정한다.
- 패링/카운터 계열은 짧은 유효 시간 안에 서버 확정 판정으로 성공 여부를 판단한다.
- 서버는 모든 스킬을 `원형`, `부채꼴`, `직사각형`, `이동 경로`, `투사체 라인` 중 하나로 환원해 처리한다.

## 권장 입력 예시

- `좌클릭`: 기본 공격
- `우클릭`: 패링 또는 가드
- `Q`, `E`, `R`, `F`: 액티브 스킬 4개
- `Space`: 회피 또는 이동기
- `Tab`: 아이덴티티 스킬
- `Z`, `X`: 소모품

숫자키 중심 배치는 `WASD`와 손이 멀어지므로 1차 기본안에서는 우선순위를 낮춘다.

## 초기 스킬 지급

- 캐릭터 생성 완료 직후 선택한 무기 타입의 스킬 4종(Q/E/R/F 슬롯 각 1개)을 레벨 1로 자동 지급한다.
- 초기 지급 스킬은 각 무기의 기본형 스킬로, 성장 전 기준값을 가진다.
- 아이덴티티 스킬(`Tab`)은 별도 지급 없이 무기 타입에 따라 자동 활성화된다.

### 스킬 초기 슬롯 구성

| 슬롯 | 지급 방식 | 비고 |
|---|---|---|
| Q, E, R, F | 캐릭터 생성 시 자동 지급 (무기별 기본형 4종) | 스킬북 별도 획득 불필요 |
| Tab (아이덴티티) | 무기 타입에 따라 자동 활성화 | 별도 습득 없음 |
| Z, X (소모품) | 비어있음. NPC 상점 또는 드롭으로 보충 | |

## 주요 흐름

### 스킬 습득과 성장

1. 캐릭터 생성 시 선택한 무기의 기본 스킬 4종이 자동 지급된다.
2. 마을에서 다른 무기를 장착하면 해당 무기의 기본 스킬 4종이 자동 연결된다.
3. 던전에서 스킬북을 획득하면 해당 무기 스킬의 강화 변형을 추가로 해금한다.
4. 레벨업으로 얻은 스킬 포인트를 스킬 레벨 업(1→5)과 변형 선택에 투자한다.
5. 현재 원정의 주무기에 맞춰 파밍을 이어가며 빌드를 고도화한다.

### 아이덴티티 운용

1. 전투 중 공격 적중으로 무기별 게이지를 모은다.
2. 게이지를 소모해 무기 고유의 강한 순간 성능을 발동한다.
3. 검방은 보호와 탱킹, 지팡이는 제어, 대검은 폭딜이라는 역할 차이를 만든다.

## 무기별 조작 예시

### 검방

- 진입 수단: 짧은 돌진 후 베기(`E`)로 정면을 잡는다.
- 방어 응답: `우클릭` 패링 또는 가드 유지로 위험 패턴을 받는다.
- 처벌 수단: 방패 강타(`Q`)와 내려치기(`F`)로 무력화를 누적한다.
- 실패 복구: 보호막 또는 도발(`R`)로 파티 정렬을 다시 잡는다.
- 숙련 포인트: 보스 정면에서 패링 성공 후 즉시 반격 창을 여는 리듬.

### 대검

- 진입 수단: 전진 참격(`E`)으로 간격을 좁힌다.
- 방어 응답: `우클릭` 카운터 자세로 큰 패턴을 받는다.
- 처벌 수단: 차지 강공격(`R`)과 내려찍기(`F`)로 다운 구간 폭딜을 넣는다.
- 실패 복구: 넓은 전방 베기(`Q`)로 잡몹을 정리하며 위치를 회복한다.
- 숙련 포인트: 무력화 타이밍에 맞춰 가장 큰 한 방을 안전하게 꽂는 판단.

### 지팡이

- 진입 수단: 직선 투사체(`Q`)와 장판 폭발(`E`)로 안전 거리에서 교전한다.
- 방어 응답: `우클릭` 짧은 보호막 또는 시전 유지로 대응한다.
- 처벌 수단: 긴 시전 주문(`F`)과 속성 디버프(`R`)를 다운 구간에 집중한다.
- 실패 복구: 제어 장판과 거리 유지로 파티가 다시 정렬할 시간을 번다.
- 숙련 포인트: 보스 전조가 열리는 순간에만 긴 시전을 눌러 손실 없이 폭딜을 넣는 판단.

## 제약과 예외

- 아이덴티티 예시는 방향성 정의용이며 수치와 세부 밸런스는 후속 조정 대상이다.
- 트라이포드식 변형은 빌드 다양성 확보가 목적이며 슬롯 수 자체는 크게 늘리지 않는다.
- 로그인 시 여러 무기 타입의 스킬 정보를 메모리에 올리는 방향을 전제로 한다.
- 쿨다운, 자원 검증, 동기화 방식은 구현 문서에서 확정한다.
- 범위 지정 스킬은 전체 스킬 중 소수만 사용하고, 과도한 클릭 조작이 반복되지 않게 한다.
- 무기 전환 가능 구조가 곧 `모든 무기를 같은 깊이로 즉시 운용 가능`을 뜻하지는 않는다. 프리셋과 성장 정도는 무기별로 분리한다.
