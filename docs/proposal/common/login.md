# 로그인 및 세션 관리

## 목적

계정 생성부터 캐릭터 선택 화면까지의 흐름을 정의한다.

## 핵심 규칙

- 계정은 ID(영문+숫자 4~20자) + 비밀번호로 생성한다.
- 로그인 성공 시 서버는 세션 토큰을 발급한다.
- 세션 토큰은 메모리에서만 유지하며 DB에 저장하지 않는다.
- 중복 로그인 시 기존 세션을 강제 종료하고 새 세션을 생성한다.
- 비정상 종료 후 재접속 시 동일 토큰으로 복구를 시도한다.
- 세션 유효 시간은 비활성 30분으로 한다.

## 주요 흐름

### 1. 계정 생성

1. 클라이언트가 ID와 비밀번호를 서버에 전송한다.
2. 서버는 ID 중복 여부를 검사한다.
3. 중복이 없으면 계정 정보를 DB에 저장한다.
4. 결과를 클라이언트에 응답한다.

### 2. 로그인

1. 클라이언트가 ID와 비밀번호를 전송한다.
2. 서버는 ID/PW를 검증한다.
3. 검증 성공 시 세션 토큰을 생성하고 메모리에 저장한다.
4. 기존 세션이 있으면 기존 세션을 강제 종료한 뒤 새 세션을 등록한다.
5. 세션 토큰과 함께 캐릭터 목록을 조회해 클라이언트에 전달한다.

### 3. 캐릭터 선택 화면

1. 서버로부터 받은 캐릭터 목록을 표시한다 (최대 3캐릭터).
2. 플레이어는 기존 캐릭터를 선택하거나 새 캐릭터 생성을 선택한다.
3. 캐릭터 선택 완료 시 해당 캐릭터로 마을 입장을 요청한다.

### 4. 로그아웃

1. 클라이언트가 로그아웃을 요청한다.
2. 서버는 메모리의 세션 토큰을 무효화한다.
3. 연결을 종료한다.

## 제약과 예외

- 로그인 실패 5회 시 60초 잠금 처리한다.
- 계정 삭제 기능은 포트폴리오 범위 밖이다.
- 패킷 암호화, OAuth 같은 보안 고도화는 구현 단계에서 확정한다.
- 비밀번호 저장 방식(해시 알고리즘 등)은 구현 단계에서 확정한다.
