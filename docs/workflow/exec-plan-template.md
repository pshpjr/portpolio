# Exec Plan 템플릿

exec-plan 파일을 새로 만들 때 이 템플릿을 복사해서 사용한다.

- 위치: 해당 앱/영역의 `docs/exec-plans/active/`
- 파일명: `YYYYMMDD-작업명.md` (예: `20260403-harness-single-source.md`)
- 완료 시: `completed/`로 이동하고 active INDEX.md에서 항목 제거

---

```markdown
# Exec Plan: {작업명}

## 목표

한 문장으로 이 작업이 달성해야 할 것.

## 범위

- 수정/생성할 파일 또는 디렉터리 목록
- 영향받는 레이어 또는 도메인

## 완료 기준

- [ ] 구체적인 기능/구조 기준 1
- [ ] 구체적인 기능/구조 기준 2
- [ ] (서버 작업 시) 빌드 통과
- [ ] (서버 작업 시) check_layers.py 통과
- [ ] (문서 작업 시) doc_check.py 통과

## 진행 상황

- [ ] 항목 1
- [ ] 항목 2

## 설계 결정

작업 중 내린 결정과 이유를 여기에 기록한다.
나중에 `docs/design/`으로 이동할 수 있다.
```

---

## 작성 원칙

- **목표**는 "무엇을"이 아니라 "왜"에 가깝게 쓴다. 구현 세부사항은 범위에 담는다.
- **완료 기준**은 체크 가능한 사실로 쓴다. "잘 동작한다"는 기준이 아니다.
- **진행 상황**은 작업 중 실시간으로 갱신한다. 완료된 항목은 `[x]`로 표시.
- **설계 결정**은 왜 그 선택을 했는지 이유를 남긴다. 코드 주석으로 담기 어려운 맥락이 대상이다.
- plan이 범위를 벗어나면 본문을 먼저 수정하고 작업을 계속한다.

## 크기 기준

| 변경 규모 | exec-plan |
|---|---|
| 300줄 미만, 단일 파일 | 선택 사항 |
| 300줄 이상 또는 여러 파일 | 필수 |
| 레이어·도메인 경계를 넘는 작업 | 필수 |
| 하네스(에이전트/스킬) 추가·변경 | 필수 |
