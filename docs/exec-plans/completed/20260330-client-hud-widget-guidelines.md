# Exec Plan: Client HUD Widget Guidelines

## 목표

- 언리얼 클라이언트에서 HUD와 여러 `UUserWidget` 베이스를 어떻게 구성할지 베스트 프랙티스를 문서화한다.
- 인벤토리, 장비, 창고, 스킬 핫바처럼 성격이 다른 UI를 어떤 계층으로 나누는지 기준을 남긴다.
- 체력/소모 자원, 캐릭터 정보, 스킬트리, 퀘스트, 업적, 메일처럼 화면군이 다른 UI의 배치 기준을 정리한다.
- 입력 키와 UI 슬롯 개념을 분리하는 원칙, 위젯 책임 분리, 데이터 바인딩 방식을 정리한다.

## 범위

- 루트 문서:
  - `docs/design/client-hud-widget-guidelines.md`
  - `docs/exec-plans/active/INDEX.md`

## 완료 기준

- [x] HUD 하나와 여러 위젯 베이스를 함께 운용하는 구조를 설명한다.
- [x] 언제 별도 최상위 위젯으로 분리하고 언제 화면용 컨테이너 위젯 아래로 묶는지 기준을 제시한다.
- [x] `AHUD`, `APlayerController`, `UUserWidget`, `UActorComponent`의 책임 분리를 설명한다.
- [x] 인벤토리/장비/창고/스킬 핫바 기준 예시 구조를 포함한다.
- [x] 입력 매핑과 슬롯 도메인 enum을 분리하는 이유를 설명한다.
- [x] 체력/소모 자원, 캐릭터 정보, 스킬트리, 퀘스트, 업적, 메일 화면의 권장 구조를 포함한다.

- [x] exec-plan 작성 및 범위 확정
- [x] HUD/UI 베스트 프랙티스 문서 작성

## 설계 결정 기록

- 이 문서는 특정 기능 구현 절차보다 클라이언트 UI 아키텍처 가이드에 가까워 `docs/design/` 아래에 둔다.
- 저장소에 아직 `client/docs/` 체계가 없으므로, 상위 설계 문서로 관리해 이후 클라이언트 문서 체계가 생겨도 참조점으로 쓸 수 있게 한다.
- 인벤토리/장비/창고만 따로 문서화하기보다, 플레이어가 실제로 접하는 HUD와 메타 화면군 전체를 한 문서에서 함께 보는 편이 구조 판단에 더 유리하다.
