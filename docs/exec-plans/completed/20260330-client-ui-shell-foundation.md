# Exec Plan: Client UI Shell Foundation

## 목표

- 언리얼 클라이언트에서 인벤토리 초안과 이어지는 파티/동의/알림 UI 골격을 추가한다.
- 파티 가입 창, 소형 파티 상태창, 공용 동의 팝업, 업적 달성 노티를 블루프린트 친화적인 베이스 클래스로 정리한다.
- 파티/노티 데이터가 서버 동기화 레이어 없이도 UI 프로토타이핑에 바로 연결되도록 공통 타입을 정의한다.

## 범위

- 클라이언트 코드:
  - `client/Source/client/Public/UI/`
  - `client/Source/client/Private/UI/`
  - 필요 시 `client/Source/client/client.Build.cs`
- 루트 문서:
  - `docs/exec-plans/active/INDEX.md`

## 완료 기준

- [x] 파티 가입 목록 데이터를 담는 타입과 가입 요청 이벤트를 가진 위젯 베이스가 생긴다.
- [x] 파티원 얼굴, 레벨, 체력 정보를 담는 타입과 소형 상태창용 위젯 베이스가 생긴다.
- [x] 던전 입장 동의 같은 공용 동의 팝업을 위한 큐/응답 API가 생긴다.
- [x] 업적 달성 토스트를 위한 큐/표시 API가 생긴다.
- [x] 주요 타입과 메서드가 블루프린트에서 접근 가능한 형태로 노출된다.
- [x] 최소 1회 이상 빌드 또는 정적 점검 시도를 하고 결과를 기록한다.

## 설계 결정 기록

- 현재 `client/`에는 전용 AGENTS/exec-plan 체계가 없으므로 이번 UI 골격도 루트 `docs/exec-plans/active/`에서 관리한다.
- 위젯은 곧바로 완성형 레이아웃을 강제하지 않고 `UUserWidget` 베이스와 데이터 바인딩 API만 제공해 UMG 블루프린트 프로토타이핑 속도를 높인다.
- 파티 가입, 동의 팝업, 업적 노티는 모두 서버 이벤트를 UI에 반영하는 성격이 강하므로 공통 데이터 타입과 큐 중심 흐름으로 시작한다.
- 파티 상태창은 개별 파티원 엔트리와 전체 상태 묶음 두 레벨로 나눠, 작은 HUD와 상세 패널이 같은 데이터를 재사용할 수 있게 한다.
- 빌드 점검은 처음엔 샌드박스의 `UnrealBuildTool` 로그 파일 이동 권한 문제로 실패했지만, 권한 상승 후 `UE_5.7`의 `Build.bat clientEditor Win64 Development ...` 실행이 최종 성공했다.
