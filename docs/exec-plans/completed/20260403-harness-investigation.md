## 설계 결정 기록

- 이번 작업은 구현이 아니라 루트 문서/운영 체계 조사이므로 `docs/exec-plans/active/`에서 관리한다.
- 조사 단계에서는 파일 수정 범위를 exec-plan 문서와 active index로 제한하고, 우선 현황과 리스크를 명확히 파악한다.
- 현재 하네스 문서는 `.codex/`와 `.claude/`에 역할 정의를 이중 보관하고 있어, 생성 규칙이나 단일 소스가 없으면 드리프트 위험이 크다.
- 하네스 스킬은 Claude 중심의 팀 도구(`TeamCreate`, `SendMessage`, `TaskCreate`)를 기본값으로 설명하지만, 현재 Codex 실행 환경은 `spawn_agent` 중심이라 설계 문서와 실행 플랫폼 간 정렬이 필요하다.
- 저장소는 문서 라우팅은 잘 정리되어 있지만, 루트 앱 목록과 실제 디렉터리 상태가 완전히 일치하지 않는다. `client/`는 이미 존재하지만 루트 진입 문서는 아직 “예정”으로 적고 있고, `shared/`는 실제 공용 자산이 있으나 루트 앱 경계 설명에 포함되지 않는다.
