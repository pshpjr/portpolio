# [server 폴더 기준 모노레포 재배치

## 목표
현재 서버 프로젝트 루트 구성을 `server/` 하위로 이동해, 이후 `client/`, `dummyserver/`를 같은 리포지토리에서 병렬 관리할 수 있는 기본 레이아웃을 만든다.

## 범위
- 루트의 서버 프로젝트 파일/폴더를 `server/` 하위로 이동
- 루트 `AGENTS.md`를 모노레포 진입점으로 재구성
- 새 구조에 맞는 최소 문서/경로 안내 반영

## 완료 기준
- [ ] 서버 프로젝트 핵심 파일이 `server/` 하위에 존재
- [ ] 루트에 모노레포용 `AGENTS.md`가 존재
- [ ] `server/AGENTS.md` 및 관련 문서 경로가 정상 동작
- [ ] 주요 루트 구조를 재확인해 누락 없이 이동됨

## 진행 상황

- [ ] 구조 검증

## 설계 결정 기록
- 서버 프로젝트는 현재 상대 경로 기반 문서/빌드 구성이 많으므로, 관련 루트 파일들을 함께 `server/`로 이동해 내부 상대 경로를 최대한 보존한다.
- 모노레포 루트에서는 개별 앱 디렉터리(`server/`, 향후 `client/`, `dummyserver/`)로 진입하도록 안내하는 얇은 `AGENTS.md`를 둔다.
